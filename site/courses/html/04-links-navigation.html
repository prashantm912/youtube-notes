Module: Links and Navigation

A hyperlink is the web's original interaction — one attribute, href, turns text into a doorway. This module maps every part of the anchor, from URL schemes to focus order, and points you to the references that go deep.

Knowledge Library → Courses → HTML → Links and Navigation

What this module covers

Links are what make separate documents into a web. This module covers the anchor element and the URL schemes its href can point at, the attributes that shape a link's behaviour and safety (target, rel, download, ping, referrerpolicy), fragment navigation with smooth-scroll control, and the focus and accessibility rules that decide whether a link actually works for keyboard and screen-reader users. Learn these before forms or interactive components — navigation underlies both.

Topics

The anchor element and href schemes

The <a> element plus its href creates a hyperlink to anything a URL can address: another page, a file, a location on the current page, an email address, or a phone number. The value can be an absolute URL, a relative path resolved against the current document, a bare fragment (#id), or a non-HTTP scheme — mailto: opens a mail client, tel: starts a call on capable devices, and blob:/data: point at in-memory or inline content. An anchor with no href is inactive: it is not a link, not focusable, and exposes no link role.

Why it matters: this is the most-used element on the web, and interviewers probe whether you understand that "link" means "a URL-addressable target", not just "another web page". Relative-versus-absolute path resolution and the special schemes are exactly the details that separate a memorised answer from an understood one.

Link attributes: target, rel, download, ping, referrerpolicy

Attributes turn a plain link into a controlled navigation. target="_blank" opens the destination in a new browsing context; rel declares the relationship and carries the security and SEO keywords — noopener (severs the new page's access to your window.opener), noreferrer (noopener plus stripping the Referer header), nofollow, sponsored, me. download asks the browser to save the target instead of navigating (same-origin, with an optional suggested filename). ping fires a background POST to listed URLs for click tracking, and referrerpolicy controls how much referrer information leaks to the destination.

Why it matters: modern engines apply noopener implicitly for target="_blank", but explaining why the vulnerability existed — reverse tabnabbing — and why you still write rel for older browsers and privacy is a classic security-flavoured HTML question in Indian product-company rounds.

<a href="https://partner.example"
   target="_blank"
   rel="noopener noreferrer">Partner site (new tab)</a>

<a href="/reports/q2.pdf" download="quarter-2-report.pdf">Download report</a>

Fragments, in-page navigation, and scroll behaviour

A fragment link (href="#section-id") navigates to the element whose id matches — or to a <a name> in legacy pages — and updates the URL so the position is shareable and bookmarkable. href="#top" or bare href="#" jumps to the document start. Text fragments (#:~:text=) go one step further and link to a phrase even when the author added no id. The jump itself is instant by default; scroll-behavior: smooth in CSS animates it, and scroll-margin-top on the target reserves space so a sticky header does not cover the heading you landed on.

Why it matters: single-page anchors, tables of contents, and "back to top" controls are everywhere, and the fixed-header-hides-the-target bug is a real interview scenario — knowing the CSS-only fix (scroll-margin-top) rather than reaching for JavaScript signals current knowledge.

html { scroll-behavior: smooth; }        /* animate in-page jumps */
:target { scroll-margin-top: 5rem; }     /* clear a sticky header */

tabindex, focus order, and accessible links

Links are focusable by default and sit in the natural tab order that follows source order. tabindex adjusts that: 0 makes a normally non-focusable element focusable at its document position, -1 makes it focusable only by script (useful for moving focus after navigation), and any positive value is an anti-pattern that yanks the element to the front of the tab order and breaks the visual-to-keyboard mapping. Accessible links also need discernible text — screen-reader users pull up a list of links out of context, so "click here" and "read more" are useless; the link's own words must describe the destination. A skip link at the top of the page lets keyboard users jump past repeated navigation to the main content.

Why it matters: accessibility is now a scored requirement in enterprise and government-adjacent work in India, and "what does tabindex do" plus "why is positive tabindex bad" is one of the most reliable accessibility screens in front-end interviews.

Interview questions

Frequently asked in Indian front-end and full-stack interviews — links and accessibility are staples of the HTML and DOM screening round:

Beginner What does the href attribute do, and what happens to an <a> with no href?

href supplies the destination the link points at — a URL, a path, a fragment, or a scheme like mailto:. An anchor with no href is an inactive anchor: it renders but is not a link, is not keyboard-focusable, and has no link role. Use it only when you plan to add the target later.

Beginner How do you open a link in a new tab, and what must you add?

target="_blank". Historically you also added rel="noopener noreferrer" to stop the new page reaching back through window.opener and to withhold the referrer. Modern browsers imply noopener for target="_blank", but writing it keeps older browsers safe and documents intent.

Beginner What do mailto: and tel: links do?

mailto: opens the user's default mail client with the address (and optional subject/body query) pre-filled; tel: hands a phone number to the device's dialer, which is why it is common on mobile. Both are just URL schemes the href understands — no JavaScript required.

Beginner How do in-page (fragment) links work?

href="#faq" scrolls to the element with id="faq" and appends #faq to the URL so the position is bookmarkable. A bare # or #top jumps to the top of the document. The target also matches the CSS :target pseudo-class.

Beginner What does the download attribute do?

It tells the browser to save the linked resource instead of navigating to it, and its value (optional) suggests a filename. It is honoured for same-origin URLs (and blob:/data:); cross-origin downloads are restricted for security.

Intermediate rel="noopener" vs rel="noreferrer" — the difference?

noopener sets the opened page's window.opener to null so it cannot script your page, and lets it run in a separate process. noreferrer does everything noopener does and suppresses the Referer header so the destination cannot see where the click came from. Use noreferrer when you also want referrer privacy.

Intermediate When should you use a link versus a button?

Use <a href> for navigation — going to a URL, a page, or a fragment (it belongs in history, can be opened in a new tab, and works without JavaScript). Use <button> for an in-page action — submitting, toggling, opening a dialog. Faking a button with a link (or vice versa) breaks keyboard behaviour and the accessibility role.

Intermediate A fixed header hides the heading after an anchor jump. How do you fix it without JavaScript?

Add scroll-margin-top (roughly the header's height) to the scroll target, so the browser stops short of the sticky header. Pair it with scroll-behavior: smooth on the root if you want the jump animated. Both are pure CSS — no scroll-offset scripting needed.

Intermediate What do tabindex="0" and tabindex="-1" mean, and why avoid positive values?

0 inserts a normally non-focusable element into the natural tab order at its DOM position; -1 makes an element focusable by script (element.focus()) but not by tabbing. Positive values force the element ahead of everything else in tab order, decoupling keyboard flow from visual order and creating a maintenance trap — the guidance is to use only 0 and -1.

Intermediate What is a skip link and why does it matter?

A skip link is usually the first focusable link on the page (href="#main"), visually hidden until focused, that lets keyboard and screen-reader users jump past repeated navigation straight to the main content. Without it, every page begins with the same tab slog through the menu — a common WCAG finding in accessibility audits.

Pro What is reverse tabnabbing, and how is it mitigated today?

When a page opens another with target="_blank", the opened page historically received a live window.opener reference and could redirect the original tab to a phishing look-alike while the user was away on the new tab — reverse tabnabbing. Mitigation: rel="noopener" nulls that reference; browsers now apply it implicitly for target="_blank". Defence-in-depth still writes rel for legacy engines.

Pro What makes link text accessible, and what is wrong with "click here"?

Screen-reader users often navigate by a generated list of links pulled out of context, so each link's own text must describe its destination. "Click here" and "read more" collapse to a meaningless list; instead write "Read the refund policy". If design forces terse text, supply context with aria-label or visually-hidden text — but real, descriptive link text is the first choice, and it also helps SEO and everyone scanning the page.
