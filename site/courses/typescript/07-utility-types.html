Module: Utility Types

TypeScript ships two dozen ready-made type transformers — the difference between rewriting a type by hand and reshaping it in a single line. This module maps each one to where it earns its keep and points you at the resources that teach it fully.

What this module covers

Utility types are generic type aliases the compiler makes globally available so you can derive one type from another instead of duplicating it: making fields optional, picking or dropping keys, filtering a union, or reading a function's return type. They are the everyday payoff of generics, mapped types and conditional types — this module orients you on each family and its intent, and leaves the exhaustive reference to the linked docs. Under the hood every one is either a mapped type, a distributive conditional type, or a compiler intrinsic, so knowing how they are built is what turns memorised names into real fluency.

Topics

Object mapping: Partial, Required, Readonly, Record, Pick, Omit

These reshape an object type by rewriting its property set. Partial makes every field optional (patch and update payloads), Required reverses that, and Readonly marks each field immutable at the type level. Record<Keys, Type> builds a fresh object type from a key set and a value type; Pick keeps a named subset of keys and Omit keeps everything except the named keys.

Why it matters: these are the most-used utilities in real code — a form's draft state is a Partial, a lookup table is a Record, a public DTO is an Omit of internal fields. They are all shallow mapped types, so knowing that Omit<T, K> is really Pick<T, Exclude<keyof T, K>> is a common follow-up.

interface User { id: number; name: string; email: string; }

type Draft   = Partial<User>;             // every field optional
type Full    = Required<Draft>;           // reverses Partial
type Frozen  = Readonly<User>;            // no reassignment
type ById    = Record<number, User>;      // { [k: number]: User }
type Card    = Pick<User, "id" | "name">; // keep two keys
type Public  = Omit<User, "email">;       // drop one key

Union filtering: Exclude, Extract, NonNullable, NoInfer

Where the first group reshapes objects, this group filters unions. Exclude<U, X> drops the union members assignable to X; Extract<U, X> keeps only those; NonNullable<T> is the special case that removes null and undefined. All three are distributive conditional types — they apply to each union member independently, which is why they operate member-by-member rather than on the union as a whole.

Why it matters: filtering discriminated-union variants and stripping nullability show up constantly, and interviewers use them to probe whether you understand distributive conditional types. NoInfer<T> (TypeScript 5.4) is the newer sibling: it blocks a position from being used as a source of inference, so a fallback argument must match a type inferred elsewhere instead of widening it.

type Status = "idle" | "loading" | "done" | "error";

type Settled = Exclude<Status, "idle" | "loading">;  // "done" | "error"
type Busy    = Extract<Status, "loading" | "x">;      // "loading"
type Name    = NonNullable<string | null | undefined>; // string

// NoInfer pins inference to the first argument (TS 5.4)
function paint<C extends string>(all: C[], fallback: NoInfer<C>) {}
paint(["red", "green"], "blue"); // error: "blue" is not one of the colours

From functions and constructors: Parameters, ReturnType, InstanceType, Awaited, ThisType

This family reads types out of callables. Parameters<F> gives a function's argument tuple, ReturnType<F> its result, InstanceType<C> the type a constructor produces, and Awaited<T> recursively unwraps a Promise the way await does. ThisType<T> is the odd one out — a marker that supplies contextual this typing inside object-literal methods (used by mixin-style config objects).

Why it matters: these keep derived types in sync with an implementation — annotate a wrapper with ReturnType<typeof original> and it tracks the source automatically. Under the hood ReturnType is a conditional type with infer (T extends (...a: any) => infer R ? R : any), which is exactly why "how would you implement ReturnType?" is a favourite interview probe.

function make(id: number, name: string) { return { id, name }; }

type Args = Parameters<typeof make>;   // [number, string]
type Out  = ReturnType<typeof make>;   // { id: number; name: string }
type Inst = InstanceType<typeof Date>; // Date
type Data = Awaited<Promise<Promise<number>>>; // number

String literal types: Uppercase, Lowercase, Capitalize, Uncapitalize

Four intrinsic types transform string literal types at the type level. Uppercase and Lowercase convert the whole string; Capitalize and Uncapitalize touch only the first character. They distribute over string unions, so applying one to a union transforms every member.

Why it matters: combined with template literal types they generate precise API surfaces from a small seed — deriving onClick-style handler names from an event union, or enforcing a naming convention in a config key. Unlike the other utilities these are compiler intrinsics: you cannot write them in userland TypeScript, which is a neat detail to know for a senior round.

type Event   = "click" | "hover";
type Handler = `on${Capitalize<Event>}`; // "onClick" | "onHover"

type Loud  = Uppercase<"hello">;   // "HELLO"
type Quiet = Lowercase<"HELLO">;   // "hello"
type Lower = Uncapitalize<"Hello">; // "hello"

Interview questions

Utility types are a TypeScript screening favourite in Indian interviews — from service-company rounds to product-company deep dives; here are the most-asked, easy to hard:

Beginner What are utility types in TypeScript?

Built-in generic type aliases the compiler exposes globally that derive a new type from an existing one — Partial, Pick, ReturnType and so on. They save you from duplicating a type when you only need a reshaped version of it.

Beginner Difference between Pick and Omit?

Pick<T, K> keeps only the keys you list; Omit<T, K> keeps everything except the keys you list. They are inverses — Omit<T, K> is effectively Pick<T, Exclude<keyof T, K>>.

Beginner What does Partial do, and where would you use it?

It makes every property of a type optional. Typical uses: update/patch payloads where a caller sends only the changed fields, and merging user-supplied options over defaults.

Beginner What does Record<K, V> construct?

An object type whose keys are K and values are V, for example Record<string, number>. With a finite key union such as Record<"light" | "dark", Theme> the compiler even enforces that every key is present.

Beginner How is Readonly<T> different from JavaScript const?

Readonly<T> marks each property immutable in the type system (a compile-time, shallow guarantee that is erased at runtime). const only prevents reassigning a binding — you can still mutate the object it points to.

Intermediate Exclude vs Extract?

Both filter a union. Exclude<U, X> removes members assignable to X; Extract<U, X> keeps only members assignable to X. They are complementary, and both are distributive conditional types.

Intermediate Why is Partial called "shallow", and how would you make it deep?

Partial only makes top-level keys optional; nested objects stay as they were. A deep version needs a recursive mapped type — type DeepPartial<T> = { [K in keyof T]?: DeepPartial<T[K]> } — which is why the built-ins are best understood as one-level mapped types.

Intermediate How does ReturnType work under the hood?

It is a conditional type using infer: roughly type ReturnType<T> = T extends (...a: any) => infer R ? R : any. The infer R captures the result type in the true branch. Parameters works the same way, inferring the argument tuple.

Intermediate What problem does NoInfer<T> solve?

Introduced in TypeScript 5.4, it stops a position from contributing to generic inference. Wrapping a default or second argument in NoInfer forces it to match the type inferred from the other arguments instead of widening the type parameter to accept it.

Intermediate What does Awaited<T> do and why was it added?

It recursively unwraps Promise and thenable types, modelling what await yields — Awaited<Promise<Promise<number>>> is number. It was added in 4.5 to fix nested-promise unwrapping and to make Promise.all infer correctly.

Pro Implement Omit yourself using other utilities.

type MyOmit<T, K extends keyof any> = Pick<T, Exclude<keyof T, K>>. It computes the keys to keep with Exclude, then re-picks them. A mapped-type variant with key remapping — { [P in keyof T as P extends K ? never : P]: T[P] } — is the modern alternative and shows command of the as clause.

Pro Why does Exclude distribute over unions, and how do the string types like Uppercase differ?

Exclude is a distributive conditional type: when its checked type is a naked union, the condition applies to each member separately, which is precisely the filtering behaviour. Uppercase, Lowercase, Capitalize and Uncapitalize also distribute over string unions, but they are compiler intrinsics — implemented in the compiler and not expressible in userland TypeScript. Knowing which utilities are "written in TS" versus "compiler magic" is a senior-level distinction.
