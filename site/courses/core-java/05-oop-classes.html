Module: OOP: Classes and Objects

A class is a blueprint and an object is what you stamp from it — this module walks the mechanics of turning source into live objects, and the visibility rules that decide who is allowed to touch them.

Knowledge Library → Courses → Core Java → OOP: Classes and Objects

What this module covers

This module is how Java turns declarations into running objects: the class body, constructors and the this reference, the static world of class-level state, and initializer blocks with their exact run order. It then turns to who may see what — the four access modifiers, packages as both namespace and access boundary, and how import and import static bring names into scope. Encapsulation is decided here, and every later OOP topic assumes you have it cold.

Topics

Classes/objects, constructors, this, static, initializer blocks

A class is the blueprint; an object is an instance created with new. The class body holds fields (state), methods (behavior) and constructors (initialization). A constructor shares the class name, has no return type, and runs once per object to set up its fields; overloaded constructors differ by parameter list, and this(...) chains one constructor to another. Inside an instance method or constructor, this is a reference to the current object — most often used to disambiguate a field from a same-named parameter.

Why it matters: the static modifier binds a field or method to the class itself — one shared copy, callable without an instance (Math.max). A static initializer block runs once when the class is loaded; an instance initializer block is copied into every constructor and runs before the constructor body. Knowing the precise order — static fields and blocks at class load, then per object the instance fields and blocks, then the constructor body — is one of the most reliable interview probes in this whole track.

class Counter {
    static int created = 0;            // one shared copy for the class
    final int id;                      // one per instance
    { created++; }                     // instance initializer: runs before the ctor body
    Counter()       { this(-1); }      // delegate to the other constructor
    Counter(int id) { this.id = id; }  // this.id is the field, id is the parameter
}

Access modifiers, packages, imports (incl. static imports)

Access modifiers control visibility. There are four member levels: public (visible everywhere), protected (same package plus subclasses), package-private (no keyword — same package only), and private (same class only). At the top level a class may be only public or package-private. Packages group related types, prevent name clashes and form the unit of package-private access; the package statement must be the first line of a file and mirrors the directory layout. import brings a type's simple name into scope, while import static brings in static members so you can write max(a, b) instead of Math.max(a, b).

Why it matters: encapsulation — expose the minimum surface — is graded in almost every interview. Keep fields private, publish behavior, and widen access only with a reason. Reversed-domain package names (com.example.app) keep type names globally unique, and static import reads cleanly for math or assertion DSLs but, overused, hides where a name comes from — hence the standard "use sparingly" caveat.

package com.example.orders;          // first statement; matches the folder path

import java.util.List;               // import a single type
import static java.lang.Math.max;    // import a static member

public class Order {                 // public: visible everywhere
    private long id;                 // private: this class only
    protected String status;         // package + subclasses
    int lineCount;                   // package-private (no keyword)
    int biggest(int a, int b) { return max(a, b); } // static import in use
}

Interview questions

Class fundamentals dominate Indian screening rounds and product-company deep dives alike — these are the ones that keep coming back:

Beginner What is a constructor, and how is it different from a method?

A special block that initializes a new object: it carries the class name, has no return type, and runs when new is called. Unlike a method it cannot be invoked on an existing object, is not inherited, and cannot be abstract, static, or final.

Beginner What is the default (no-argument) constructor?

If you declare no constructor at all, the compiler supplies a no-argument one whose body just calls super(). Declare any constructor and that free one disappears — a frequent cause of "no default constructor" failures with frameworks that instantiate by reflection.

Beginner What does the static keyword mean for a field or method?

The member belongs to the class, not to any instance — one shared copy, reachable as ClassName.member without creating an object. A static method has no this, so it cannot touch instance fields directly.

Beginner Name the four access levels from least to most visible.

private (same class), package-private / default (same package), protected (same package plus subclasses), public (everywhere). Top-level classes may be only public or package-private.

Beginner What is a package and how do you declare one?

A namespace that groups related types. You declare it with a package com.x.y; statement as the first line of the file, mirroring the folder path. It prevents name clashes and defines the default (package-private) access boundary.

Intermediate What does this do, and what is constructor chaining with this()?

this references the current object — commonly to separate a field from a same-named parameter (this.x = x). this(args) calls another constructor of the same class and must be the first statement, letting constructors reuse each other's initialization instead of duplicating it.

Intermediate Static vs instance initializer blocks — when does each run?

A static block runs once when the class is loaded, in source order, together with static field initializers. An instance block runs on every object creation, after the superclass constructor and before the current constructor body — the compiler copies it into each constructor, which is how sibling constructors share setup code.

Intermediate Can a top-level class be private or protected? What about static?

No. A top-level class may be only public or package-private. private, protected and static apply to nested classes; putting static or private on a top-level class is a compile error.

Intermediate What is package-private (default) access and when is it useful?

No modifier means the member is visible only within the same package. It is the "internal" scope Java gives you for free — ideal for helper classes and members shared with collaborators in the package but hidden from the public API.

Intermediate What is a static import, and when should you avoid it?

import static java.lang.Math.max; lets you write max(...) unqualified. It reads well for Math, shared constants and test assertions such as assertEquals, but overuse hides which class a name comes from and invites ambiguity — the official guidance is to use it sparingly.

Pro Walk through the exact initialization order when new Sub() runs, where Sub extends Base.

First, class loading (once each): Base's static field initializers and static blocks, then Sub's, in top-down source order. Then, for the object: Base's instance field initializers and instance blocks, Base's constructor body, then Sub's instance field initializers and instance blocks, then Sub's constructor body. Superclass initialization always completes before the subclass's begins.

Pro Why is calling an overridable method from a constructor dangerous?

The base constructor runs before the subclass's fields are initialized, yet a virtual call dispatches to the subclass override — which then observes its own fields still at defaults (null or 0), producing partial-state bugs and surprise NPEs. This is the "this-escape" hazard (JDK 21 added a this-escape lint warning). Call only private, final, or static methods from a constructor, or move that work into a method the caller runs after construction.
