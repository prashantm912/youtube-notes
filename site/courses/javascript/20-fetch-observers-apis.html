Module: Browser: Fetch, Observers and APIs

The browser is a runtime, not just a document viewer. This module surveys the JavaScript APIs that pull data over the network, react to the page and device without polling, and drive motion — pointing you to the official reference for each and the questions interviewers build on top of them.

What this module covers

Everything in the previous module lived inside the document; this one covers the JavaScript APIs that connect it to the network, the browser session, and the device. You will meet the Fetch API with URL/URLSearchParams, the History/Location/Navigation trio behind client-side routing, the four Observer APIs that report change asynchronously, and a cluster of platform APIs — Web Animations, View Transitions, File/Blob, TextEncoder, clipboard and geolocation. The common thread: each is asynchronous, promise- or callback-based, and increasingly gated behind secure contexts and user permission.

Topics

Fetch (Request/Response/Headers), URL/URLSearchParams

The Fetch API is the promise-based interface for HTTP in the browser: fetch(input, init) returns a Promise that resolves to a Response, and the request/response are modelled by three objects — Request, Response, and Headers (a multimap with get/set/append). Alongside it, the URL interface parses and builds URLs component by component, and its searchParams (a URLSearchParams) reads and writes the query string with automatic percent-encoding — so you never hand-concatenate ?a=&b= again.

Why it matters: fetch replaced XMLHttpRequest and underpins every SPA data call, but its single most-tested gotcha is that it only rejects on a network failure — an HTTP 404 or 500 still resolves, so you must check response.ok yourself. That, plus building a URL safely and streaming the body, are near-universal in Indian front-end screening rounds.

const url = new URL("https://api.example.com/search");
url.searchParams.set("q", "hello world");   // space is auto-encoded

const res = await fetch(url, { headers: { Accept: "application/json" } });
if (!res.ok) throw new Error(`HTTP ${res.status}`); // 404/500 do NOT reject
const data = await res.json();

History/Location, Navigation API

Location (reached as window.location) reads and writes the current URL and can assign, replace, or reload. The History API lets a single-page app change the address bar without a full reload via pushState/replaceState, restoring Back/Forward through the popstate event. The newer Navigation API is the modern successor: it routes every navigation — link clicks, form posts, Back/Forward, and programmatic calls — through one navigate event whose intercept() handler you use to render the new route.

Why it matters: client-side routing is the backbone of React Router, Vue Router and friends, and interviewers probe whether you understand what those libraries do underneath. The Navigation API exists precisely because the History API has sharp edges — pushState fires no event, intercepting a navigation was awkward, and scroll/focus restoration was manual. It reached Baseline in 2026, so knowing it signals you keep current.

// History API: change the URL without reloading, then handle Back
history.pushState({ page: 2 }, "", "/list?page=2");
window.addEventListener("popstate", (e) => render(e.state));

// Navigation API: intercept every navigation in one place
navigation.addEventListener("navigate", (e) => {
  if (!e.canIntercept) return;
  e.intercept({ handler: async () => renderRoute(e.destination.url) });
});

Observers (Intersection/Mutation/Resize/Performance)

The four Observer APIs share one pattern — construct with a callback, observe() targets, receive batched entries asynchronously — but watch different things. IntersectionObserver reports when an element enters or leaves the viewport (or an ancestor), powering lazy-loading, infinite scroll, and scroll-triggered animation. MutationObserver watches the DOM tree for added/removed nodes and attribute changes. ResizeObserver fires when an element's own box changes size — the JavaScript route to container-query-style behaviour. PerformanceObserver surfaces entries from the performance timeline (marks, measures, LCP, long tasks).

Why it matters: every one of these replaces an expensive pattern — a scroll/resize listener plus getBoundingClientRect() running synchronously on the main thread, or the long-deprecated Mutation Events. Because observer callbacks are batched and run off the hot path, they scale to hundreds of targets without jank. "How would you lazy-load images?" and "why is IntersectionObserver faster than a scroll handler?" are staple performance questions.

const io = new IntersectionObserver((entries) => {
  for (const entry of entries) {
    if (entry.isIntersecting) {
      entry.target.src = entry.target.dataset.src; // lazy-load
      io.unobserve(entry.target);                  // one-shot: stop watching
    }
  }
}, { rootMargin: "200px" });                        // start loading early

document.querySelectorAll("img[data-src]").forEach((img) => io.observe(img));

Web Animations API, View Transitions JS, File/Blob, TextEncoder, clipboard/geolocation

This topic is the platform grab-bag that rounds out a working browser developer. The Web Animations API runs keyframe animations from JavaScript through Element.animate() with full playback control (play, pause, reverse, seek). The View Transitions API — document.startViewTransition(updateDOM) — snapshots the old and new states and animates between them with CSS. Blob is immutable binary data and File extends it (what an <input type="file"> or a drag-drop yields). TextEncoder turns a string into a UTF-8 Uint8Array. And navigator.clipboard and navigator.geolocation read from the system clipboard and the device's location.

Why it matters: these APIs show up constantly in real UI work — file uploads, copy-to-clipboard buttons, "find near me", polished page transitions — and they carry the module's recurring security lesson. The Clipboard and Geolocation APIs only work in a secure context (HTTPS) and require user permission or a user gesture, which is exactly the kind of "why doesn't this work on my HTTP dev server?" question interviewers like to pose.

// Animate a DOM change with a view transition + the Web Animations API
if (document.startViewTransition) {
  document.startViewTransition(() => updateTheDOM());
}
element.animate(
  [{ opacity: 0 }, { opacity: 1 }],
  { duration: 200, easing: "ease-out" }
);

await navigator.clipboard.writeText("copied!"); // secure context + gesture

Interview questions

The browser-API questions Indian interviews return to most, from front-end screening rounds to product-company system-design discussions:

Beginner What does fetch() return, and does an HTTP 404 reject the promise?

fetch() returns a Promise that resolves to a Response. It rejects only on a network-level failure (offline, DNS, CORS block). A 404 or 500 still resolves successfully — so you must check response.ok (or response.status) and throw yourself. This is the single most common fetch gotcha.

Beginner How do you send JSON in a POST request with fetch?

Pass an init object: method: "POST", a Content-Type: application/json header, and body: JSON.stringify(payload). The body must be a string (or Blob/FormData/etc.), not a plain object — forgetting JSON.stringify is a frequent bug. Read the reply with await res.json().

Beginner Why use URL/URLSearchParams instead of string-concatenating a query string?

URLSearchParams percent-encodes keys and values automatically (spaces, &, non-ASCII), preserves multiple values per key, and serialises with toString(). Hand-building "?q=" + input breaks on any special character and is an injection risk. new URL(base) also validates and normalises the whole address.

Beginner How do you cancel an in-flight fetch request?

Create an AbortController, pass its signal in the fetch init, and call controller.abort() to cancel — the fetch promise rejects with an AbortError. The standard use is a timeout or cancelling a stale request when the user types a new search term.

Beginner What is IntersectionObserver used for?

It asynchronously reports when a target element enters or leaves the viewport (or a specified ancestor). Typical uses: lazy-loading images, infinite scroll, firing animations when a section scrolls into view, and measuring ad/impression visibility — all without a scroll-event handler.

Intermediate Why was the Navigation API introduced when we already had the History API?

The History API has gaps: pushState/replaceState fire no event, so a router can't observe its own navigations centrally; intercepting a user navigation before it happens was awkward; and scroll/focus restoration was manual. The Navigation API routes every navigation through one navigate event with intercept(), exposes ongoing transitions, and manages state and scroll restoration — a purpose-built SPA routing primitive.

Intermediate How does IntersectionObserver improve performance over a scroll listener?

A scroll handler runs synchronously on the main thread on every scroll tick, and computing visibility with getBoundingClientRect() forces layout — janky with many elements. IntersectionObserver computes intersections off the main thread and invokes your callback asynchronously in batches only when a threshold is crossed, so cost is tied to actual visibility changes, not scroll frequency.

Intermediate When would you reach for ResizeObserver versus a window resize listener?

A window.resize listener only fires on viewport changes and tells you nothing about an element resized by content, flexbox, or a sibling. ResizeObserver fires when a specific element's box changes for any reason, delivers the new dimensions, and avoids the resize-loop feedback problems — the right tool for component-level responsive logic (JS-side container queries).

Intermediate What is the difference between a File and a Blob?

A Blob is immutable raw data with a size and a MIME type. A File is a Blob with extra metadata — name and lastModified — produced by <input type="file">, drag-and-drop, or the File System APIs. Anything that accepts a Blob (fetch body, URL.createObjectURL, FileReader) accepts a File too.

Intermediate Why do the Clipboard and Geolocation APIs sometimes silently fail?

Both require a secure context (HTTPS or localhost), so they do nothing on a plain-HTTP page. They are also permission-gated: geolocation prompts the user, and clipboard writes generally need a user gesture and read needs permission. Add Permissions-Policy and cross-origin iframe restrictions and you have the usual causes of "works locally, breaks in the iframe".

Pro Explain CORS as it applies to fetch: simple vs preflighted requests, and credentials.

A cross-origin fetch is allowed only if the server returns Access-Control-Allow-Origin. "Simple" requests (GET/POST/HEAD with safe headers and a basic content type) go straight out and the browser checks the response header. Anything else — PUT/DELETE, custom headers, application/json — triggers a preflight OPTIONS that must be answered with matching Access-Control-Allow-Methods/-Headers. To send cookies you set credentials: "include", and then the server must return a specific origin (not *) plus Access-Control-Allow-Credentials: true. CORS is enforced by the browser, not the server.

Pro How does the View Transitions API work under the hood, and how do you degrade gracefully?

document.startViewTransition(cb) captures a snapshot of the current page, runs your cb to update the DOM, snapshots the new state, then builds a tree of ::view-transition pseudo-elements and cross-fades old to new — animatable with CSS (and named groups via view-transition-name for shared-element transitions). Because it returns a promise-bearing object and only animates when supported, you guard with if (document.startViewTransition) and otherwise just update the DOM directly — the change still happens, only without the animation. It respects prefers-reduced-motion, which you should honour.
