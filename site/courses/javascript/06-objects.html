Module: Objects

Objects are JavaScript's core data structure — arrays, functions and classes are all objects underneath. This module maps the literal syntax, the descriptor machinery beneath every property, and the Object statics you reach for daily.

Knowledge Library → Courses → JavaScript → Objects

What this module covers

A JavaScript object is a keyed collection of properties, and almost everything else in the language is one underneath. This module works through three layers: the ergonomic literal syntax (property shorthand and computed keys), the property descriptors and accessor properties that decide how each key behaves, and the Object.* static methods for inspecting, copying, transforming and locking objects.

Topics

Object literals: shorthand and computed properties

The object initializer { } is the everyday way to build objects, and modern syntax makes it compact: property shorthand lets { name } stand in for { name: name }, method shorthand drops the : function, and computed property names put an expression in square brackets so the key is decided at runtime. Keys are always strings (or Symbols); numeric keys are coerced to strings.

Why it matters: shorthand and computed keys are the difference between clumsy and idiomatic code, and they appear constantly in React props, reducers and config objects. Reading someone else's { [action.type]: handler } and knowing exactly what key it produces is basic fluency an interviewer expects.

const key  = "role";
const name = "Ada";
const user = {
  name,                                    // shorthand for name: name
  [key]: "admin",                          // computed key -> user.role
  greet() { return "hi, " + this.name; },  // method shorthand
};

Property descriptors, getters/setters and defineProperty

Every property is more than a value: it carries hidden attributes — writable, enumerable and configurable — packaged as a property descriptor. Object.getOwnPropertyDescriptor reads them and Object.defineProperty sets them, letting you create read-only or non-enumerable keys that a plain assignment never could. Assignment (obj.x = 1) creates properties that default all three flags to true; defineProperty defaults them to false.

Why it matters: accessor properties — a get/set pair instead of a value — power computed fields, validation and lazy evaluation while looking like ordinary properties to callers. Descriptors are also how framework internals (Vue 2 reactivity, immutability helpers) hook into reads and writes, which makes them a favourite "what's under the hood" interview thread.

const account = { balance: 100 };
Object.defineProperty(account, "balance", {
  writable: false,     // reassignment silently ignored (throws in strict mode)
  enumerable: false,   // hidden from for..in and Object.keys
});

const temp = {
  celsius: 21,
  get fahrenheit() { return this.celsius * 9 / 5 + 32; },  // accessor
  set fahrenheit(f) { this.celsius = (f - 32) * 5 / 9; },
};

Object statics: keys, values, entries, assign, freeze, create, groupBy

Because objects have no methods of their own for bulk work, the utilities live as static methods on the Object constructor. Object.keys/values/entries turn an object into arrays you can iterate or feed to Object.fromEntries; Object.assign and object spread copy own enumerable properties; Object.freeze, seal and preventExtensions lock objects down to varying degrees; Object.create builds an object with a chosen prototype (including null for a clean dictionary).

Why it matters: these are the daily tools for transforming, cloning and protecting state, and the newest addition — Object.groupBy (Baseline 2024) — buckets an iterable by a callback key in one line, replacing hand-rolled reduce accumulators. Knowing which methods skip inherited or non-enumerable keys, and that freeze is shallow, is where the interview depth lives.

const prices = { apple: 2, pear: 3 };
Object.keys(prices);       // ["apple", "pear"]
Object.entries(prices);    // [["apple", 2], ["pear", 3]]

const frozen = Object.freeze({ pi: 3.14 });   // shallow; nested objects still mutable
const dict = Object.create(null);             // no prototype -> safe lookup table

Object.groupBy([1, 2, 3, 4], n => n % 2 ? "odd" : "even");
// { odd: [1, 3], even: [2, 4] }

Interview questions

Most-asked object questions in Indian interviews, from screening rounds to product-company deep dives:

Beginner What are the ways to create an object in JavaScript?

Object literal {} (most common), new Object(), a constructor function with new, ES6 class, and Object.create(proto) to set the prototype explicitly. The literal is preferred for plain data; Object.create(null) makes a prototype-less dictionary.

Beginner Dot notation vs bracket notation — when must you use brackets?

Both read/write properties. Use brackets when the key is dynamic (held in a variable), computed at runtime, not a valid identifier (has spaces or starts with a digit), or is a Symbol. obj.user is the literal key "user"; obj[user] uses the value of the variable user as the key.

Beginner What are property shorthand and computed property names?

Shorthand lets you write { x } instead of { x: x } when the key matches an in-scope variable. Computed names put an expression in brackets inside the literal — { [prefix + id]: value } — so the key is evaluated when the object is built rather than being a fixed string.

Beginner Object.keys vs Object.values vs Object.entries — what does each return?

All return arrays of the object's own enumerable string-keyed properties: keys gives the key names, values the values, entries an array of [key, value] pairs. entries plus Object.fromEntries is the standard map-over-an-object idiom.

Beginner How do you copy an object, and what is shallow vs deep?

Shallow copy: { ...obj } or Object.assign({}, obj) — top-level keys are copied but nested objects are still shared by reference. Deep copy: structuredClone(obj) (built-in, handles most types) or, with caveats, JSON.parse(JSON.stringify(obj)) which drops functions, undefined and Dates.

Intermediate What is a property descriptor? Name the attributes.

A descriptor is the metadata object behind a property. Data properties have value, writable, enumerable, configurable; accessor properties replace value/writable with get and set. Read one with Object.getOwnPropertyDescriptor, set one with Object.defineProperty. Assignment defaults the flags to true; defineProperty defaults them to false.

Intermediate Difference between a data property and an accessor property?

A data property stores a value. An accessor property has no stored value — it runs a get function on read and a set function on write, so it can compute a result or validate input while looking exactly like a normal property to callers. A property is one or the other, never both at once.

Intermediate Object.freeze vs Object.seal vs Object.preventExtensions?

preventExtensions: no new properties, but existing ones stay writable and deletable. seal: also blocks deletion and reconfiguring, but values can still change. freeze: seal plus makes values non-writable — fully immutable at the top level. All three are shallow.

Intermediate for...in vs Object.keys — what's the difference?

for...in walks own and inherited enumerable string keys, so it can pick up prototype pollution — you guard it with Object.hasOwn(obj, key). Object.keys returns only own enumerable keys as an array. Prefer Object.keys/entries for plain-data iteration.

Pro Object.assign vs spread — are they identical?

Both copy own enumerable properties. Key differences: Object.assign mutates the target and triggers setters on it (assignment semantics), while spread always creates a fresh object and defines plain data properties (no target setters). Neither copies the prototype or non-enumerable keys, and both invoke getters on the source, copying the resulting value.

Pro Why is Object.freeze only shallow, and how do you deep-freeze?

Freeze only touches the object's own properties; a property holding a nested object stores a reference, and the referenced object is untouched, so its contents stay mutable. Deep-freeze by recursing: freeze the object, then walk Object.values (or keys) and recursively freeze any value that is an object or function, guarding against cycles.

Pro What does Object.create(null) give you that {} does not?

An object with no prototype, so it has none of the inherited members (toString, hasOwnProperty, and crucially no __proto__ accessor). That makes it a safe hash map for untrusted keys — you avoid collisions with inherited names and one class of prototype-pollution attack. The trade-off: you must call Object.hasOwn/Object.keys statically since the instance has no methods of its own.
