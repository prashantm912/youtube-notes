Module: Destructuring and Modern Syntax

The same three dots that pull values out of arrays and objects also push them back in — and that pair of patterns, plus a couple of readability niceties, is the syntax modern JavaScript is actually written in.

Knowledge Library → Courses → JavaScript → Destructuring and Modern Syntax

What this module covers

Destructuring and spread are two halves of one idea: binding patterns that unpack arrays and objects into variables, and spread/rest syntax that expands or gathers them back again. Together with numeric separators they form the everyday "modern syntax" you will read in every React component, Node service, and interview whiteboard. The payoff is code that is both shorter and clearer — once you learn the small rules: defaults that trigger only on undefined, copies that are shallow, and rest that must come last.

Topics

Object/array destructuring (nested, defaults, rest)

Destructuring is a binding-pattern syntax that unpacks values in a single statement: array destructuring reads elements by position (and works on any iterable), while object destructuring reads properties by name. Object patterns can rename ({ id: userId }), reach into nested structures ({ user: { city } }), and supply defaults ({ page = 1 }); a trailing ...rest element or property gathers whatever is left into a fresh array or object.

Why it matters: this is the shape of modern JavaScript — React state and props (const [count, setCount] = useState(0)), named imports, function parameter objects, and the temp-free variable swap. Two rules reliably earn interview points: a default fills in only when the value is undefined, never when it is null, and destructuring into already-declared variables needs wrapping parentheses so the leading { is not read as a block.

const [first, , third = 0, ...rest] = [10, 20];
// first = 10, third = 0 (default used), rest = []

const { id, name: title, meta: { tags = [] } = {} } = post;
// title = post.name; tags defaults to [] when meta or tags is absent

Spread in calls/arrays/objects, numeric separators

Spread syntax (...) is destructuring in reverse: it expands an iterable into individual elements, or an object's own enumerable properties into a new object literal. Use it to pass an array as separate arguments (Math.max(...nums)), to concatenate or clone arrays ([...a, ...b]), and to copy-and-merge objects ({ ...defaults, ...overrides }), where a later key overrides an earlier one. The same three dots on the parameter side become rest parameters, collecting a variable number of arguments into a real array — the modern successor to the array-like arguments object.

Why it matters: spread is how you write immutable updates — copy, then change — that Redux reducers and React state setters depend on; the catch interviewers probe is that the copy is shallow, so nested objects stay shared and need structuredClone for a true deep copy. Numeric separators are the module's small readability win: underscores placed between digits (1_000_000) that the engine ignores, legal anywhere except at the start or end of a literal, two in a row, or right after a leading zero.

const merged = { ...defaults, ...overrides };  // later keys win
function sum(...nums) { return nums.reduce((a, b) => a + b, 0); }
const budget = 1_000_000;  // 1000000 — the underscores are cosmetic

Interview questions

Most-asked destructuring and modern-syntax questions in Indian interviews, from screening rounds to product-company deep dives:

Beginner What is destructuring, and what problem does it solve?

A syntax that unpacks array elements (by position) or object properties (by name) into separate variables in one statement. It replaces repetitive access like const city = user.address.city and manual indexing, making code shorter and self-documenting.

Beginner How do you swap two variables without a temporary variable?

[a, b] = [b, a]. The right side builds an array from the current values, then array destructuring assigns them back in swapped order — no temp needed.

Beginner How do you give a default value while destructuring, and when does it apply?

const { page = 1 } = query. The default is used only when the property is undefined — missing or explicitly undefined. If the value is null, 0, or "", the default does not apply, which is the classic trick question.

Beginner What does the spread operator do? Give an array and an object example.

It expands an iterable or object into individual elements. [...a, ...b] concatenates two arrays, { ...a, ...b } merges two objects, and fn(...args) spreads an array into separate function arguments.

Beginner How do you shallow-copy an array or object with spread?

const copy = [...arr] or const clone = { ...obj }. It creates a new top-level container, but nested objects and arrays inside remain shared references — it is a shallow copy, not a deep one.

Intermediate Rest and spread use the same ... — how do they differ?

Position decides. On the left of an assignment or in a parameter list, ... is rest — it collects the remaining items into an array or object. On the right of an assignment, in a function call, or inside a literal, it is spread — it expands items out. Rest gathers; spread scatters.

Intermediate How do you rename a variable and destructure a nested property at once?

const { name: title, author: { id: authorId } } = post. Here name: title binds post.name to title, and the nested pattern reaches post.author.id as authorId. Note that author itself is not created as a variable — only the leaves you name are.

Intermediate How do you safely destructure a function parameter object that might be omitted?

Default the whole parameter to an empty object: function f({ id, size = 10 } = {}). Without the = {}, calling f() throws, because you cannot destructure undefined. The pattern gives you per-property defaults and a safe no-argument call.

Intermediate Merge two objects with spread — which key wins, and how does it compare to Object.assign?

In { ...a, ...b }, later sources win, so b's keys override a's. It is close to Object.assign({}, a, b), but spread always builds a new object (no mutation) and only copies own enumerable properties, whereas Object.assign mutates its first argument and invokes setters on the target.

Intermediate What are numeric separators, and what are the rules?

Underscores placed between digits for readability — 1_000_000, 0xFF_EC_DE, 1_000.50. They are purely cosmetic; the value is unchanged. The separator must sit between two digits: no leading or trailing underscore, none doubled up, and none right after a leading 0 (0_1 is a SyntaxError).

Intermediate Does array destructuring work on objects, and object destructuring on arrays?

Array destructuring works on any iterable — arrays, strings, Sets, generators — and binds by order. Object destructuring works on almost any value (primitives are boxed) and binds by property name, so const { length } = "hi" gives 2. You cannot array-destructure a plain non-iterable object: const [a] = {} throws "object is not iterable".

Pro Spread makes a shallow copy — what breaks, and how do you deep-clone?

Nested objects and arrays in the copy are the same references as the original, so mutating copy.nested.x also changes the source. For a genuine deep copy use structuredClone(obj) (built in, handles cycles and many built-in types). JSON.parse(JSON.stringify(obj)) works only for JSON-safe data — it drops functions and undefined and turns Date into a string.

Pro Why does { a } = obj throw, and how do you destructure into existing variables?

A statement that starts with { is parsed as a block, not an object pattern, so the assignment is a syntax error. Wrap the whole thing in parentheses: ({ a } = obj). This only bites when assigning to already-declared variables; with let or const the keyword makes the intent unambiguous and no parentheses are needed.
