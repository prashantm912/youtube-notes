Module: Meta-programming and Built-ins

Some of the language's most powerful tools sit one layer below everyday code: objects that intercept their own operations, serializers and cloners, a long-overdue date API, and formatting that already knows how the world writes numbers and time. This module maps each built-in to the problem it exists to solve.

What this module covers

Meta-programming is code that acts on the behaviour of other code. In JavaScript that means Proxy and Reflect for intercepting an object's fundamental operations, plus the platform's core built-ins you reach for constantly: JSON serialization with its replacer/reviver hooks and structuredClone for real deep copies, the legacy Date object together with its modern replacement Temporal, and the Intl family for locale-aware formatting. These are deep-dive interview topics precisely because the naive answer and the senior answer diverge sharply.

Topics

Proxy (all traps), Reflect

A Proxy wraps a target object and lets you intercept the fundamental operations performed on it through handler functions called traps: property reads (get), writes (set), existence checks (has), deletion (deleteProperty), key enumeration (ownKeys), descriptor access (getOwnPropertyDescriptor, defineProperty), prototype and extensibility operations, and — for callable targets — apply and construct. Any operation you don't trap passes straight through to the target. Reflect is the companion namespace: for every trap there is a same-named Reflect method with the same signature that performs the default operation, so a trap can do its custom work and then forward to Reflect to get correct default behaviour.

Why it matters: Proxy is the engine behind reactive frameworks (Vue 3's reactivity, MobX), validation and access-control layers, logging and observability wrappers, and negative-index or default-value shims. Reflect.get(target, key, receiver) is preferred over target[key] inside a trap because it threads the correct receiver, so an inherited getter sees the right this; and Reflect methods return booleans instead of throwing, which pairs cleanly with traps that must return a boolean. Interviewers probe whether you understand the invariants — a get trap must return the real value of a non-writable, non-configurable data property or the engine throws a TypeError.

const user = new Proxy({ age: 30 }, {
  set(target, key, value, receiver) {
    if (key === "age" && !Number.isInteger(value))
      throw new TypeError("age must be an integer");
    return Reflect.set(target, key, value, receiver); // default write, correct receiver
  }
});
user.age = 31;      // ok
user.age = "old";   // throws TypeError

JSON (parse/stringify, replacer/reviver), structuredClone

JSON.stringify serializes a value to a JSON string and JSON.parse does the reverse. The second argument of each is a transform hook: stringify's replacer — a function, or an allow-list array of property names — filters and rewrites values on the way out; parse's reviver runs on every key/value on the way in, the standard place to rehydrate Date objects from ISO strings. The third stringify argument, space, controls pretty-print indentation, and any object can define a toJSON() method to customize its own serialized form. The gotchas are heavily tested: undefined, functions, and symbols are dropped from objects (and become null in arrays), Map/Set serialize to {}, BigInt throws, and circular references throw.

Why it matters: JSON.parse(JSON.stringify(x)) is the old deep-clone hack, and knowing why it is wrong is the point of the question — it silently loses Date, Map, Set, and typed arrays, and blows up on cycles. structuredClone(value) is the built-in deep clone using the structured-clone algorithm: it preserves cycles, Date, RegExp, Map, Set, ArrayBuffer and typed arrays, and can transfer transferables. It cannot clone functions, DOM nodes, or an object's prototype chain (the result is a plain object). The senior answer to "how do you copy an object" separates a shallow spread, the lossy JSON round-trip, and structuredClone.

const data = { when: new Date(), tags: new Set(["a", "b"]) };

const text = JSON.stringify(data, (key, val) =>
  val instanceof Set ? [...val] : val, 2);   // replacer + pretty-print

const revived = JSON.parse(text, (key, val) =>
  key === "when" ? new Date(val) : val);      // reviver rehydrates the Date

const clone = structuredClone(data);          // keeps Date & Set intact, no round-trip loss

Date + Temporal (Instant/PlainDate/ZonedDateTime/Duration)

The legacy Date object represents a single instant as milliseconds since the Unix epoch, but it is famously error-prone: it is mutable, months are zero-indexed, parsing anything but ISO strings is implementation-defined, and it conflates wall-clock time with an exact instant while really only understanding the local zone and UTC. Temporal is the modern replacement, built as a namespace of distinct, immutable types for distinct concepts: Temporal.Instant (an exact point on the timeline), Temporal.ZonedDateTime (an instant tied to an IANA time zone), the zone-less Temporal.PlainDate / PlainTime / PlainDateTime calendar-and-wall-clock values, and Temporal.Duration (a length of time), with DST-safe arithmetic and non-ISO calendar support.

Why it matters: at TC39's March 2026 plenary Temporal advanced to Stage 4 and is slated for ECMAScript 2026 — it is done and shipping (Firefox 139, Chrome 144, Node 26). It is not yet Baseline across every browser, so MDN marks it "Limited availability" and an official polyfill remains prudent for broad support, but new code should target Temporal rather than Date. Being able to name the four core type families and explain why an immutable, zone-aware API removes an entire class of bugs is exactly the kind of "do you keep up with the language" signal interviewers look for.

const now = Temporal.Now.zonedDateTimeISO("Asia/Kolkata"); // an instant in a zone
const start = Temporal.PlainDate.from("2026-08-01");
const shifted = start.add({ months: 1, days: 10 });         // immutable → new value

const gap = start.until("2026-12-25", { largestUnit: "day" }); // a Temporal.Duration
gap.days;                                                   // whole days between the dates

Intl (NumberFormat/DateTimeFormat/Collator/RelativeTimeFormat/Segmenter)

The Intl namespace is the built-in internationalization toolkit: locale-aware formatting and comparison with no external libraries. The workhorses are Intl.NumberFormat (decimals, currency, units, percentages, compact notation), Intl.DateTimeFormat (locale date/time formatting, ranges, and formatToParts), Intl.Collator (locale-correct string comparison and sorting), Intl.RelativeTimeFormat (phrasing such as "3 days ago"), and Intl.Segmenter (locale-correct grapheme, word, and sentence segmentation — the right way to count user-perceived characters or split words in scripts written without spaces). The family also includes PluralRules, ListFormat, DisplayNames, and DurationFormat.

Why it matters, especially for an Indian audience: hand-rolled formatting breaks the moment locale enters the picture. Manual comma grouping cannot produce the Indian numbering system's lakh/crore grouping (1,23,456), naive concatenation fumbles the ₹ symbol placement and right-to-left scripts, and String.length miscounts emoji and combining marks because it counts UTF-16 code units, not characters. Intl.NumberFormat("en-IN") yields correct digit grouping and currency, Collator sorts Devanagari properly, and Segmenter counts what a user would call a character. Reaching for Intl instead of bespoke string code is a recurring code-review and interview signal.

new Intl.NumberFormat("en-IN", { style: "currency", currency: "INR" })
  .format(123456.5);                 // "₹1,23,456.50"  (lakh/crore grouping)

new Intl.NumberFormat("en-IN", { notation: "compact" })
  .format(12300000);                 // "1.2Cr"

const seg = new Intl.Segmenter("hi", { granularity: "grapheme" });
[...seg.segment("👍🏽")].length;       // 1 user-perceived character, not 4 code units

Interview questions

Most-asked meta-programming and built-ins questions in Indian interviews, from screening rounds to product-company deep dives:

Beginner What is a Proxy in JavaScript?

An object that wraps a target and intercepts fundamental operations on it via traps defined on a handler — get, set, has, deleteProperty, ownKeys, apply, construct, and more. Operations you don't trap fall through to the target. It powers reactivity systems, validation, and logging wrappers.

Beginner What values does JSON.stringify silently drop or fail on?

undefined, functions, and symbols are dropped from objects and become null inside arrays. Map/Set serialize to {}, BigInt throws a TypeError, and a circular reference throws. Dates are serialized via their toISOString() (through toJSON) but come back as strings, not Date objects.

Beginner What is the difference between the replacer and the reviver?

The replacer is the second argument to JSON.stringify — a function or an allow-list array that filters/transforms values on the way out. The reviver is the second argument to JSON.parse — a function run on each key/value on the way in, typically to turn ISO strings back into Date objects.

Beginner Why is the classic Date object considered problematic, and what replaces it?

It is mutable, months are zero-indexed, parsing non-ISO strings is implementation-defined, and it only really models UTC and the local zone. Temporal replaces it with immutable, zone- and calendar-aware types (Instant, ZonedDateTime, PlainDate/PlainTime/PlainDateTime, Duration).

Intermediate How do you deep-clone an object, and what is wrong with JSON.parse(JSON.stringify(x))?

Use structuredClone(x). The JSON round-trip loses Date, Map, Set, typed arrays, and undefined, converts everything to plain objects, and throws on cycles or BigInt. structuredClone preserves those types and handles circular references, but still cannot clone functions, DOM nodes, or prototypes.

Intermediate What is Reflect and why call it inside a Proxy trap?

Reflect is a namespace whose methods mirror the Proxy traps one-for-one and perform the default operation. Inside a trap you forward to it — e.g. Reflect.get(target, key, receiver) — so the correct receiver reaches inherited getters/setters, and because Reflect returns booleans (rather than throwing) it slots straight into traps like set and defineProperty that must return a boolean.

Intermediate Name several Proxy traps and what each intercepts.

get (property read), set (write), has (the in operator), deleteProperty (delete), ownKeys (Object.keys/enumeration), getOwnPropertyDescriptor and defineProperty (descriptors), getPrototypeOf/setPrototypeOf, and for callable targets apply (calling) and construct (new). Thirteen traps map to the object's internal methods.

Intermediate How do you format an amount as Indian Rupees with correct lakh/crore grouping?

new Intl.NumberFormat("en-IN", { style: "currency", currency: "INR" }).format(amount). The en-IN locale applies the Indian grouping (1,23,456 rather than 123,456) and prepends the ₹ symbol. Never hand-roll the comma placement — the grouping is not uniform every three digits.

Intermediate Difference between Temporal.Instant, Temporal.PlainDateTime, and Temporal.ZonedDateTime?

Instant is an exact point on the global timeline (like an epoch timestamp), with no calendar or zone. PlainDateTime is a calendar date plus wall-clock time with no zone — "8 a.m. on 1 Aug" in the abstract. ZonedDateTime ties the two together via an IANA time zone, so it knows the exact instant and survives DST transitions.

Intermediate Why does str.length give the "wrong" count for emoji, and how does Intl.Segmenter help?

String.length counts UTF-16 code units, so an emoji built from surrogate pairs and modifiers (like a skin-toned thumbs-up) counts as several units. Intl.Segmenter with granularity: "grapheme" iterates user-perceived characters, giving the count and slicing a human would expect; with "word" it also splits languages without spaces (Thai, Japanese) correctly.

Pro What invariants must Proxy get/set traps respect?

The engine enforces consistency with the target. A get trap must return the actual value of a non-writable, non-configurable data property; a set trap cannot report success when writing to such a property; has cannot hide a non-configurable own property; and ownKeys must include all non-configurable own keys. Violating an invariant throws a TypeError — you cannot lie about a locked-down property.

Pro What is Temporal's standardization status, and would you ship it in production today?

Temporal reached Stage 4 at TC39's March 2026 meeting and is part of ECMAScript 2026 — the spec is final. It ships natively in Firefox 139, Chrome 144, and Node 26, but is not yet Baseline across all engines (MDN flags "Limited availability"). So: use it directly where you control the runtime, and load the official Temporal polyfill for broad browser support.

Pro Compare a spread copy, the JSON round-trip, and structuredClone for copying data.

Spread ({...obj} / [...arr]) is a one-level shallow copy — nested objects stay shared. The JSON round-trip is deep but lossy: no Date/Map/Set/typed arrays/undefined, no cycles, everything flattened to plain objects. structuredClone is deep and type-faithful (Dates, Maps, Sets, typed arrays, cycles) but drops functions, DOM nodes, and prototypes. Choose by how deep the data goes and which types it contains.
