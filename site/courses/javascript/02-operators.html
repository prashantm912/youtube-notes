Module: Operators

Operators are where JavaScript's coercion rules, short-circuiting and precedence quietly decide what your expressions actually evaluate to — the small syntax that produces the biggest interview surprises.

Knowledge Library → Courses → JavaScript → Operators

What this module covers

Every symbol that combines or transforms values: arithmetic and comparison, the logical trio (&& / || / ??) with optional chaining, the assignment family including the logical-assignment operators, and the remaining operators — ternary, comma, in, instanceof, delete, void, spread/rest and bitwise. The recurring themes are coercion, short-circuit evaluation and precedence, which is why operators dominate output-prediction rounds.

Topics

Arithmetic, exponentiation and comparison operators

Arithmetic operators (+ - * / %) and exponentiation (**) do the maths, but two behaviours bite: binary + becomes string concatenation the moment either side is a string, and ** is right-associative (2 ** 3 ** 2 is 2 ** 9, not 8 ** 2). Comparison operators (< > <= >= and the equality pair == / ===) return booleans, but relational comparisons coerce operands and string comparison runs character by character in Unicode order.

Why it matters: floating-point reality (0.1 + 0.2 is not 0.3), the NaN === NaN being false quirk, and ==-versus-=== coercion are among the most-asked screening questions. Get the mental model here and half the "predict the output" traps disappear.

2 ** 10;      // 1024  (right-associative: 2 ** 3 ** 2 === 512)
0.1 + 0.2;    // 0.30000000000000004  — floating point
"10" > "9";   // false — string compare, char by char
10 > 9;       // true  — numeric compare
NaN === NaN;  // false — the one value not equal to itself

Logical operators, nullish coalescing and optional chaining

&& and || short-circuit and return one of their operands (not a coerced boolean): || returns the first truthy value, && the first falsy one. The nullish coalescing operator ?? is the precise cousin — it falls back only on null or undefined, so legitimate falsy values like 0 and "" survive. Optional chaining ?. reads a property, index or call and short-circuits the whole chain to undefined the instant a link is nullish.

Why it matters: value ?? default versus value || default is a daily correctness decision (a 0 quantity or empty search string must not be overwritten), and ?. plus ?? together are the modern, safe way to read config and API responses — a pairing interviewers now expect you to reach for.

const count = 0;
count || 5;   // 5  — || treats 0 as falsy
count ?? 5;   // 0  — ?? only replaces null/undefined
user?.address?.city ?? "N/A";   // safe deep read with a fallback

Assignment and logical assignment operators

Plain assignment = is an expression that evaluates to the assigned value, which is why a = b = c chains. The compound operators (+= -= *= /= %= **= and the bitwise ones) modify in place. The three logical-assignment operators added in ES2021 — &&=, ||=, ??= — carry the short-circuit semantics into assignment: x ??= y assigns only when x is nullish, x ||= y only when x is falsy, x &&= y only when x is truthy, and each evaluates x just once.

Why it matters: opts.mode ??= "dark" replaces the verbose "set a default if missing" idiom, and knowing that these operators short-circuit (they skip the assignment, and any setter, when the condition fails) is a common intermediate follow-up.

let total = 10;
total += 5;            // 15
const opts = {};
opts.mode ??= "dark";  // set only when null/undefined
flag ||= true;         // assign only when falsy
count &&= count + 1;   // assign only when truthy

Ternary, spread/rest, bitwise and the other operators

The remaining operators are a grab-bag worth knowing precisely. The conditional (ternary) ?: is JavaScript's only three-operand operator and the expression form of if/else. Spread/rest (...) is one token with two jobs — spread expands an iterable into arguments, array or object literals (a shallow copy); rest gathers the leftovers into an array. Then come in (property exists in an object or its prototype chain), instanceof (a constructor's prototype appears in the chain), delete (removes an own property), void (evaluates and yields undefined), the comma operator (evaluates left-to-right, returns the last), and the bitwise set (& | ^ ~ << >> >>>), which coerce operands to 32-bit integers.

Why it matters: spread/rest and the ternary are everywhere in modern code and React; instanceof versus typeof, and bitwise tricks like n | 0 to truncate, are classic gotcha questions. The umbrella MDN guide below documents comma, delete, void and bitwise together with full precedence.

const nums = [1, 2, 3];
Math.max(...nums);               // 3   — spread
const [first, ...others] = nums; // rest -> others = [2, 3]
const merged = { ...a, ...b };   // object spread (shallow)
age >= 18 ? "adult" : "minor";   // ternary
5 & 1;                           // 1   — bitwise AND

Interview questions

Most-asked operator questions in Indian interviews, from output-prediction screens to product-company deep dives:

Beginner Difference between == and ===?

=== (strict) compares value and type with no coercion; == (loose) coerces operands to a common type first, so 0 == "", 1 == true and null == undefined are all true. Rule of thumb: use === by default; the one common exception is x == null to catch both null and undefined.

Beginner Predict the output: "5" + 3, "5" - 3, and 3 + 4 + "5".

"5" + 3 is "53" — binary + concatenates when either side is a string. "5" - 3 is 2 — - has no string meaning, so both sides coerce to numbers. 3 + 4 + "5" is "75" — left-to-right: 3 + 4 is 7, then 7 + "5" concatenates to "75".

Beginner What does the ** operator do, and is it left- or right-associative?

** is exponentiation — 2 ** 10 is 1024, equivalent to Math.pow(2, 10) but also working on BigInt. It is right-associative: 2 ** 3 ** 2 is 2 ** (3 ** 2) = 2 ** 9 = 512. A negative base like -2 ** 2 is a syntax error — you must write (-2) ** 2.

Beginner ?? versus || — when do they differ?

|| falls back on any falsy left side (0, "", false, NaN, null, undefined); ?? falls back only on null or undefined. So 0 || 5 is 5 but 0 ?? 5 is 0. Use ?? when 0 or an empty string are valid values you must keep.

Beginner Rewrite an if/else that sets a discount, using the ternary operator.

The ternary condition ? exprIfTrue : exprIfFalse is the expression form of if/else: const discount = isMember ? 0.1 : 0;. Because it is an expression it can sit inside a template literal or a return, unlike an if statement. Keep it to one condition — nested ternaries hurt readability.

Intermediate What does optional chaining return, and does it short-circuit the rest of the chain?

If the value before ?. is null or undefined, the expression stops and evaluates to undefined instead of throwing; otherwise it continues. So a?.b.c where a is undefined yields undefined and never touches .c. It works for property access ?., dynamic keys ?.[k] and calls fn?.(). It is a read guard, not an assignment target.

Intermediate With && and ||, what value is actually returned — a boolean?

No — they return one of the operands, not a coerced boolean. || returns the first truthy operand (or the last if all are falsy); && returns the first falsy operand (or the last if all are truthy). That is why a && a.b guards access and value || "default" supplies a fallback. Both short-circuit, so the right side is not evaluated when the result is already decided.

Intermediate Difference between ||= and ??= ; give a case where they diverge.

x ||= y assigns when x is falsy; x ??= y assigns only when x is null/undefined. For let n = 0; n ||= 10; you get 10 (0 is falsy), but n ??= 10 leaves 0. Both short-circuit — if the condition fails, no assignment happens and any property setter is skipped, unlike a plain x = x ?? y.

Intermediate Spread and rest use the same ... token — how do you tell them apart, and does spread deep-copy?

Position decides: ... in a call, array literal or object literal spreads (expands) a value; ... as the last function parameter or on the left of a destructuring pattern is rest (gathers into an array). Spread makes a shallow copy — { ...obj } and [ ...arr ] copy one level, so nested objects are still shared by reference. Use structuredClone for a deep copy.

Intermediate in operator versus hasOwnProperty / Object.hasOwn?

"k" in obj is true if the property exists anywhere on the object or its prototype chain; Object.hasOwn(obj, "k") (and the older obj.hasOwnProperty("k")) check own properties only. So "toString" in {} is true but Object.hasOwn({}, "toString") is false. A property set to undefined still satisfies in; only delete makes it false.

Intermediate instanceof versus typeof — and when does instanceof surprise you?

typeof returns a primitive type string; instanceof checks whether a constructor's prototype is in an object's prototype chain. Surprises: typeof null is "object", and typeof [] is "object" (use Array.isArray). instanceof fails across realms — an array from another iframe is not instanceof your Array — and returns false for primitives ("hi" instanceof String is false).

Pro Why does a ?? b || c throw a SyntaxError, and how do you fix it?

The language forbids mixing ?? directly with && or || without parentheses, precisely because their precedence is non-obvious and silent bugs would follow. It is a parse-time error, not a runtime one. Fix by making intent explicit: (a ?? b) || c or a ?? (b || c). Parenthesised, they combine freely.

Pro Bitwise gotchas: what do n | 0 and ~~n do, and why do bitwise ops break above 2^31?

Bitwise operators coerce their operands to signed 32-bit integers, so n | 0 and ~~n both truncate a number toward zero (a fast Math.trunc for in-range values). But once a number exceeds ±2^31 − 1 the 32-bit conversion overflows and wraps — 2 ** 31 | 0 is -2147483648 — so these tricks silently corrupt large integers. For values beyond 32 bits use Math.trunc, or BigInt with its own bitwise operators.
