Module: Pipes

A pipe is the small word after a vertical bar that turns raw data into what the user should actually read — and the pure/impure line beneath it is where interviewers separate template-writers from framework-thinkers.

Knowledge Library → Courses → Angular Basics → Pipes

What this module covers

Pipes transform a value for display right inside the template — value | pipeName:args — without cluttering the component class. This module orients you on the built-in pipes and locale-aware formatting, the reactive async pipe, the pure-vs-impure distinction that governs when a pipe re-runs, and how to write your own custom pipes with parameters while keeping change detection cheap.

Topics

Built-in pipes and locale formatting

A pipe is a template operator: the value on the left of | is passed to the pipe, which returns a transformed value for rendering only — the underlying data is untouched. Angular's @angular/common ships around a dozen: DatePipe, CurrencyPipe, DecimalPipe, PercentPipe, JsonPipe, the case pipes (uppercase/lowercase/titlecase), SlicePipe, KeyValuePipe and AsyncPipe. Parameters follow with colons, and pipes chain left to right.

Why it matters: date, currency and number pipes format according to locale rules, so the same template renders correctly for different regions instead of hand-rolled string juggling; json is the fastest way to dump an object while debugging a template.

{{ today | date:'dd MMM yyyy' }}      <!-- 19 Jul 2026 -->
{{ price | currency:'INR':'symbol' }} <!-- ₹1,299.00 -->
{{ name | titlecase }}
{{ user | json }}                     <!-- debug dump -->

The async pipe

{{ data$ | async }} subscribes to an Observable or Promise, hands the template its latest emitted value, and — the part that earns its keep — automatically unsubscribes when the component is destroyed. It also marks the component for change detection whenever a new value arrives. Combine it with @if (data$ | async; as data) so the stream is subscribed exactly once and the unwrapped value is reused across the block.

Why it matters: it deletes the manual subscribe/unsubscribe lifecycle — and the memory leaks that come from forgetting to unsubscribe — making it the backbone of reactive templates over HttpClient and signals-adjacent RxJS. It is the single most-asked pipe in interviews.

@if (courses$ | async; as courses) {
  <course-list [courses]="courses" />
} @else {
  <p>Loading…</p>
}

Pure vs impure pipes

By default a pipe is pure: Angular calls transform() only when the input's identity changes — a new primitive value, or a new object/array reference. Mutating an array in place (a push) does not change the reference, so a pure pipe will not re-run and the view looks stale. Setting pure: false makes the pipe impure, running transform() on every change-detection cycle — which is why AsyncPipe, JsonPipe and SlicePipe are impure.

Why it matters: purity is a performance contract. Pure pipes are memoized and can be shared across usages; impure pipes can fire hundreds of times per second. Explaining why an in-place array change doesn't update a pure filter pipe — and the fix (replace the reference, or accept impurity deliberately) — is a staple Indian-interview question.

@Pipe({ name: 'filterActive', pure: false }) // runs every change-detection pass
export class FilterActivePipe implements PipeTransform { /* … */ }

Writing custom pipes and passing parameters

A custom pipe is a class decorated with @Pipe({ name: 'x' }) (standalone by default in current Angular) that implements PipeTransform's single transform(value, ...args) method. The first argument is the value on the left of the bar; each colon-separated parameter in the template becomes the next argument, so defaults and multiple parameters work exactly like a normal function.

Why it matters: a custom pipe encapsulates reusable, unit-testable formatting behind a name you read in the template — far clearer than repeating the same expression, and declarative where a method call would be imperative.

@Pipe({ name: 'truncate' })
export class TruncatePipe implements PipeTransform {
  transform(value: string, limit = 20, trail = '…'): string {
    return value.length > limit ? value.slice(0, limit) + trail : value;
  }
}
// template:  {{ bio | truncate:50:'…' }}

Pipe performance considerations

A pure pipe is memoized by the framework: Angular caches the last result and re-invokes transform() only when an argument's identity changes. A method called directly in the template has no such cache — it re-runs on every change-detection pass, whether or not its inputs moved. That is the core reason to prefer a pure pipe over a template method for any non-trivial transformation.

Why it matters: keep transform() cheap and side-effect free, push heavy work into services, reach for pure: false only when you truly need per-cycle evaluation, and pair pipes with OnPush so the whole subtree only checks when references change. This is the performance half of the custom-pipes topic interviewers probe after the syntax.

{{ items | expensiveTransform }}  <!-- memoized: runs only when items reference changes -->
{{ expensiveTransform(items) }}   <!-- re-runs on every change-detection cycle -->

Interview questions

Most-asked pipe questions in Indian Angular interviews — the async pipe and the pure/impure distinction dominate the screening round:

Beginner What is a pipe in Angular and why use one?

A template operator that transforms a value for display: value | pipeName. It keeps formatting logic declarative and out of the component class, and — for pure pipes — is cached by Angular so it only recomputes when the input changes.

Beginner How do you pass parameters to a pipe and chain pipes?

Parameters follow the pipe name with colons: {{ today | date:'dd/MM/yyyy':'IST' }}. Chain pipes with more bars, evaluated left to right: {{ createdAt | date:'shortDate' | uppercase }}.

Beginner Name some built-in pipes you have used.

date, currency, number/decimal, percent, uppercase/lowercase/titlecase, slice, keyvalue, json and async — all from @angular/common. date, currency and number are locale-aware.

Beginner What does the async pipe do?

It subscribes to an Observable or Promise, renders the latest emitted value, marks the component for change detection on each emission, and automatically unsubscribes when the component is destroyed — so you never write manual subscribe/unsubscribe code in the template path.

Beginner How do you create a custom pipe?

Decorate a class with @Pipe({ name: 'myPipe' }) and implement PipeTransform with a transform(value, ...args) method. In current Angular the pipe is standalone by default, so import it directly into the component that uses it.

Intermediate Pure vs impure pipes — what is the difference?

A pure pipe (the default) runs transform() only when the input's identity changes — a new primitive or a new object reference. An impure pipe (pure: false) runs on every change-detection cycle. Pure is memoized and cheap; impure is powerful but expensive.

Intermediate A pure pipe filters a list. You push a new item to the array and the view doesn't update. Why, and how do you fix it?

push mutates the array in place — the reference is unchanged — so the pure pipe never re-runs. Fixes: create a new array (this.items = [...this.items, item]) so the reference changes, or make the pipe impure (last resort, watch performance). Reference-immutable updates are the idiomatic answer.

Intermediate Is the async pipe pure or impure, and why?

Impure. It holds internal subscription state and must deliver newly emitted values on every change-detection cycle, not only when its input reference changes — so it cannot be a stateless pure pipe. JsonPipe and SlicePipe are impure for related reasons.

Intermediate Pipe vs calling a method in the template — which is better for performance?

A pure pipe wins. Angular memoizes it and re-invokes only when arguments change; a template method has no cache and re-runs on every change-detection pass. For any non-trivial transformation, move it into a pure pipe.

Intermediate How does the async pipe help prevent memory leaks?

It ties the subscription to the component's lifecycle: it unsubscribes in its own ngOnDestroy when the component is destroyed. Manual subscribe() calls leak if you forget to unsubscribe (or omit takeUntilDestroyed); the async pipe removes that whole class of bug.

Pro When would you deliberately write an impure pipe, and what are the risks?

When output must reflect mutations Angular can't see by reference — e.g. filtering/sorting a mutated array, or a pipe wrapping an async source. Risks: transform() fires on every cycle, so any non-trivial work multiplies across renders. Mitigate by memoizing inside the pipe, keeping the work tiny, or redesigning around immutable updates so a pure pipe suffices.

Pro How do pipes interact with OnPush change detection and zoneless Angular?

Under OnPush a component checks only when an @Input reference changes, an event fires, or an async pipe emits — the async pipe explicitly calls markForCheck(), which is why observable-plus-async is the canonical OnPush pattern. Pure pipes fit naturally because they too depend on reference changes. In zoneless mode the same signal/async-driven marking is what schedules the pipe's re-evaluation.

Pro Your custom pipe needs to format numbers by the user's locale and inject a service. How do you do it correctly?

Pipes participate in DI, so inject dependencies via the constructor or inject() — e.g. inject LOCALE_ID or a settings service, or delegate to formatNumber/formatCurrency from @angular/common rather than reimplementing locale rules. Keep transform() deterministic in its inputs so the pure-pipe cache stays valid; if the injected state can change per cycle, that is a signal you may need impurity or a signal-based input instead.
