Module: CSS Grid

Grid is the first CSS system built for two dimensions at once — rows and columns together — and it turns layouts that once needed nested wrappers and float hacks into a handful of declarations on one container.

Knowledge Library → Courses → CSS → Grid

What this module covers

CSS Grid Layout lays out children on an explicit two-dimensional grid you define on the parent: you size the tracks (grid-template-columns/rows, the fr unit, repeat(), minmax()), place items by line number, span, or named area, let the implicit grid absorb the overflow, then align everything with the box-alignment properties and gap. Subgrid closes the loop by letting a nested grid borrow its parent's tracks. Learn it after flexbox: grid owns page and component skeletons, flexbox owns one-dimensional runs of content.

Topics

grid-template-rows/columns/areas, repeat/minmax/auto-fit/auto-fill, fr

A grid starts with display: grid and a track definition. grid-template-columns and grid-template-rows list the tracks; the fr unit distributes leftover space (so 1fr 2fr is a one-third / two-thirds split), repeat() shortens repetitive lists, and minmax(min, max) gives a track a floor and ceiling. grid-template-areas is the alternative "ascii-art" syntax: you name regions in a visual block and assign each item a grid-area, which makes the layout readable and trivially re-arrangeable at breakpoints.

Why it matters: this is the whole foundation, and one idiom — repeat(auto-fit, minmax(200px, 1fr)) — produces a responsive card wall with no media queries, which is why it is asked about constantly. Knowing what fr resolves against and how auto-fit differs from auto-fill is the difference between reciting syntax and understanding it.

.gallery {
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(200px, 1fr));
  gap: 1rem;                 /* responsive columns, zero media queries */
}

Item placement (grid-row/column, span, named lines)

Every grid has numbered lines — one more than the number of tracks — and you place an item by naming its start and end lines: grid-column: 1 / 3 puts an item between column lines 1 and 3. The span keyword says "cover N tracks" without pinning the far edge (grid-column: 1 / span 2), and negative indices count from the end, so grid-column: 1 / -1 stretches an item full width regardless of how many columns exist. You can also name lines in the template ([content-start]) and place items against those names for layouts that stay readable as tracks change.

Why it matters: placement is where grid pays off — a header, sidebar, main and footer land in exact cells with four one-line rules, and grid-column: 1 / -1 is the standard trick for a full-bleed row inside a multi-column grid. Interviewers probe whether you understand that lines, not cells, are what you address.

.feature { grid-column: 1 / -1; }      /* span every column */
.thumb   { grid-column: span 2; }      /* cover two tracks from here */
.side    { grid-row: 2 / 4; }          /* rows 2 through 3 */

Implicit grid, grid-auto-flow/rows/columns

You only define the explicit grid; when items land outside it — more children than cells, or an item placed on a line you never declared — the browser creates implicit tracks to hold them. grid-auto-rows and grid-auto-columns size those generated tracks (otherwise they are auto), and grid-auto-flow controls the auto-placement algorithm: row (default) fills across then wraps down, column fills down then across, and the dense keyword lets later small items back-fill earlier gaps.

Why it matters: most real grids are part explicit, part implicit — you declare the columns and let rows grow with the data. Understanding grid-auto-rows: minmax(100px, auto) for uniform-yet-flexible rows, and knowing that dense reorders visual output without touching DOM or tab order, are the intermediate-to-senior signals here.

Grid alignment, gap

Grid inherits the full Box Alignment module across two axes. The justify-* family works on the inline (row) axis and align-* on the block (column) axis: justify-items/align-items position every item inside its cell, justify-self/align-self override one item, and justify-content/align-content distribute the whole set of tracks when the grid is smaller than its container. The gap shorthand (with row-gap and column-gap) sets the gutters between tracks — a real property, not margins, so it never adds outer edges.

Why it matters: the classic "center anything" answer is display: grid; place-items: center — one declaration. The interview trap is *-items versus *-content: items align each box within its area, content distributes leftover space among the tracks. Confusing the two is the most common grid alignment mistake.

.center { display: grid; place-items: center; }   /* items axis */
.spread { display: grid; justify-content: space-between; } /* content axis */

Subgrid

A nested grid normally has its own independent tracks, so a child grid's columns never line up with the parent's. Setting grid-template-columns: subgrid (or rows) makes the nested element adopt the parent grid's tracks instead of defining new ones, and line names and gaps pass through too. The result: items several levels deep can still align to one shared set of lines.

Why it matters: subgrid solves the "ragged cards" problem cleanly — a row of cards whose titles, bodies and footers all align even when their content lengths differ, without JavaScript or fixed heights. It reached Baseline in 2023, so it is now a fair senior-round question about modern layout.

.cards { display: grid; grid-template-rows: auto 1fr auto; }
.card  {
  display: grid;
  grid-row: span 3;
  grid-template-rows: subgrid;   /* borrow the parent's three rows */
}

Interview questions

Most-asked grid questions in Indian interviews, from front-end screening rounds to product-company layout deep dives:

Beginner What is CSS Grid and how does it differ from Flexbox?

Grid is a two-dimensional layout system — it controls rows and columns at the same time from the container. Flexbox is one-dimensional — a single row or column of items. Rule of thumb: grid for the overall page/component skeleton, flexbox for a one-directional run of content. They compose well together.

Beginner How do you create a grid and define three equal columns?

Set display: grid on the parent, then grid-template-columns: repeat(3, 1fr). 1fr means one fraction of the available space, so three equal tracks; repeat(3, ...) is shorthand for 1fr 1fr 1fr.

Beginner What is the fr unit and what does it resolve against?

fr represents a fraction of the leftover free space in the grid container, after fixed tracks, gaps and content minimums are subtracted. So in 200px 1fr 1fr, the two fr tracks split whatever remains after the 200px column and the gaps. It is not a percentage of the whole container.

Beginner How do you put space between grid items?

The gap property (or row-gap / column-gap). It is a true gutter between tracks and adds no space on the outer edges — unlike margins, which is why it replaced the old margin-based spacing hacks. The legacy name was grid-gap.

Intermediate Explain minmax(), and how do you make a responsive grid with no media queries?

minmax(min, max) gives a track a lower and upper bound. Combined with auto-repeat — grid-template-columns: repeat(auto-fit, minmax(200px, 1fr)) — the browser fits as many at-least-200px columns as the width allows and grows them to fill the row. The layout reflows on its own as the container resizes, with no breakpoints written.

Intermediate How do you place an item to span multiple columns?

Address grid lines: grid-column: 1 / 3 spans from line 1 to line 3 (two tracks). grid-column: span 2 covers two tracks from wherever the item lands. grid-column: 1 / -1 spans every column, since -1 is the last line — the standard full-width trick. The same applies to grid-row.

Intermediate What is the implicit grid, and how do you size its tracks?

Tracks the browser creates automatically when items overflow the explicitly defined grid (more items than cells, or placement onto undeclared lines). Size them with grid-auto-rows / grid-auto-columns — e.g. grid-auto-rows: minmax(100px, auto) for rows that are at least 100px but grow with content — and steer the fill direction with grid-auto-flow.

Intermediate Difference between justify-items and justify-content in grid?

justify-items aligns each item inside its own cell along the inline axis. justify-content aligns the whole grid of tracks within the container when the grid is narrower than the container (e.g. space-between). Same split on the block axis with align-items vs align-content. Items = boxes within cells; content = tracks within the container.

Intermediate How do grid-template-areas work, and how do you leave a cell empty?

You draw the layout as strings of area names on the container ("head head" "nav main"), then give each item grid-area: head etc. Repeating a name across cells makes an item span them. A dot (.) marks an empty cell. Because the layout is visual text, you can restructure it entirely inside a media query by rewriting the strings.

Pro auto-fit vs auto-fill — what actually differs?

Both create as many repeated tracks as fit. When there are fewer items than tracks, auto-fill keeps the leftover tracks as empty columns (items stay their min size), while auto-fit collapses those empty tracks to zero width and lets the present items expand to absorb the space. Visible difference only when items do not fill the row: auto-fit stretches, auto-fill leaves gaps.

Pro What is subgrid and what problem does it solve?

grid-template-columns: subgrid / rows: subgrid makes a nested grid use its parent's tracks instead of creating its own, so deeply nested items align to one shared set of lines. It cleanly solves ragged card layouts — titles, bodies and footers across a row of cards all align despite different content lengths — with no fixed heights or JS. Baseline since 2023.

Pro What does grid-auto-flow: dense do, and what is the caveat?

dense tells the auto-placement algorithm to back-fill earlier holes with later items that fit, producing a tighter packed grid. The caveat is accessibility: it changes only the visual order, not DOM order, so the reading and tab order can diverge from what the eye sees — a WCAG concern. Use it for decorative galleries, not for content whose order carries meaning.

Pro The four-value order of grid-area, and why grid-column: 1 / -1 works?

grid-area takes row-start / column-start / row-end / column-end — row values first, a common trap versus the row / column ordering people assume. grid-column: 1 / -1 works because grid lines can be counted from the end with negative numbers: -1 is the last column line, so the item spans from the first line to the last regardless of column count. Negative indexing addresses the explicit grid only.
