The CSS Box Model

Every element you have ever seen on a web page is a rectangle. Master the four layers of that rectangle — content, padding, border, margin — and half of CSS layout debugging disappears.

Every element is a box

When the browser lays out a page, it does not think in paragraphs, buttons or images — it thinks in boxes. Each element generates a rectangular box, and the box model defines what that rectangle is made of: the content at the center, wrapped in optional padding, wrapped in an optional border, surrounded by margin that pushes other boxes away.

Why it matters: almost every "why is this element bigger than I said?" and "where is this mystery gap coming from?" bug is a box model question. The model is small enough to learn in one sitting and pays off in every layout you will ever build.

The four layers

  • Content — where text and child elements live. Controlled by width/height (in the default sizing model).
  • Padding — transparent space between content and border. Takes the element's background. Cannot be negative.
  • Border — the drawn edge: border-width, border-style, border-color. No style means no border, whatever the width says.
  • Margin — transparent space outside the border. Always transparent, can be negative, and margin: auto is the classic horizontal centering trick for block boxes with a set width.

Each layer accepts per-side values (padding-top, margin-inline-start, …) and the familiar shorthands: one value (all sides), two (vertical | horizontal), three (top | horizontal | bottom), four (top, right, bottom, left — clockwise).

box-sizing: the size math

The single most important box model decision: what does width: 200px actually mean?

  • box-sizing: content-box (the default): width sets the content only. Rendered width = width + padding + border. Your 200px box with 20px padding and a 5px border is really 250px wide.
  • box-sizing: border-box: width sets content + padding + border together. The box is exactly 200px; the content shrinks to make room.

Margin collapse: the classic surprise

Vertical margins between block boxes in normal flow do not add up — they collapse: the larger of the two wins. A paragraph with margin-bottom: 24px followed by one with margin-top: 16px is separated by 24px, not 40px.

Collapse happens in three situations: adjacent siblings, a parent with its first/last child (the child's margin "escapes" through the parent), and empty blocks collapsing through themselves. It does not happen for horizontal margins, flex items, grid items, floats, or absolutely positioned boxes.

Mystery gap above your page's first heading? That is the heading's top margin collapsing through the body. Fixes that create a new formatting context stop the escape: display: flow-root on the parent, or any padding/border on the shared edge.

Two more traps worth knowing: percentage padding and margin resolve against the parent's width (even for top/bottom — the old square-box hack), and outline draws outside the border without taking space, which is why it is the debugging tool of choice (outline: 1px solid red shifts nothing).

Interview questions

What are the four parts of the CSS box model, inside out?

Content, padding, border, margin. Content holds text/children; padding is inner space carrying the background; border is the drawn edge; margin is outer transparent space separating the box from neighbors.

An element has width: 300px, padding: 10px, border: 2px. How wide is it on screen?

Depends on box-sizing. Default content-box: 300 + 20 + 4 = 324px. With border-box: exactly 300px, content shrinks to 276px. Saying "it depends, and here's the math" is the complete answer interviewers want.

What is margin collapse and when does it NOT happen?

Adjacent vertical margins of block boxes in normal flow merge into one margin equal to the largest. It never applies horizontally, and not to flex items, grid items, floats, or absolutely positioned elements; creating a new block formatting context (e.g. display: flow-root) blocks parent–child collapse.

Why does percentage padding-top respond to the parent's width?

Spec rule: all percentage padding and margin resolve against the containing block's inline size (width). This enabled the classic aspect-ratio hack (padding-top: 56.25% for 16:9) before the aspect-ratio property existed.

outline vs border — when is each the right tool?

Border is part of the box: it takes space, affects layout and size math, styleable per side. Outline draws outside the border edge, occupies no space, can't be per-side, and follows outline-offset. Use border for design, outline for focus indicators (never remove without replacement — accessibility) and for layout debugging, since adding it shifts nothing.
