Module: Flexbox

Flexbox lays items out along one axis and hands you alignment that once took floats, clearfixes and vertical-align hacks — it is the layout mode you reach for many times a day.

Knowledge Library → Courses → CSS → Flexbox

What this module covers

Flexbox (the Flexible Box Layout) is CSS's one-dimensional layout mode: set display: flex on a container and it distributes space among its children along a main axis while aligning them on the cross axis. This module orients you on the container properties, the flex-grow/flex-shrink/flex-basis sizing algorithm on items, the four alignment properties, and gap plus the patterns flexbox solves every day. Learn it before grid — most component-level layout (nav bars, toolbars, cards, button rows) is flexbox, while grid takes over two-dimensional page structure.

Topics

Flex container (flex-direction / wrap / flow)

Declaring display: flex (or inline-flex) turns an element into a flex container and its direct children into flex items laid out along a main axis. flex-direction chooses that axis and its direction — row (default, the inline direction), column, or the *-reverse variants; the cross axis is always perpendicular. flex-wrap decides whether items stay on one line (nowrap, the default, which can overflow) or break onto new lines (wrap / wrap-reverse), and flex-flow is the shorthand that combines direction and wrap.

Why it matters: getting the axes straight is the whole game. justify-content works along the main axis and align-items along the cross axis, so flipping flex-direction silently swaps which property controls what. That axis-awareness — not memorizing property names — is what interviewers actually probe.

.toolbar {
  display: flex;
  flex-flow: row wrap;   /* flex-direction + flex-wrap */
  gap: 1rem;
}

Flex items (grow / shrink / basis, flex shorthand, order)

Each flex item has three sizing inputs: flex-basis (its starting main-size before free space is shared out), flex-grow (its share of leftover positive space), and flex-shrink (how readily it gives up space when the container is too small). The flex shorthand sets all three at once; flex: 1 expands to 1 1 0 — grow and shrink equally from a zero basis, which produces equal-width columns. The order property reorders items visually without touching the DOM.

Why it matters: the grow/shrink/basis algorithm is the single most misunderstood part of flexbox and a perennial interview topic — "what does flex: 1 actually do?" and "flex-basis vs width?" come up constantly. order is handy but decouples visual order from reading and tab order, an accessibility trap worth naming out loud.

.sidebar { flex: 0 0 240px; }  /* hold 240px; don't grow or shrink */
.main    { flex: 1 1 0; }      /* take all remaining space */

Alignment (justify-content / align-items / align-self / align-content)

Four properties do the placing: justify-content distributes items along the main axis (start, center, space-between, space-evenly…); align-items aligns them as a group on the cross axis (stretch by default, or center, flex-start, baseline); align-self overrides align-items for one individual item; and align-content distributes multiple wrapped lines along the cross axis.

Why it matters: the famous "center a div" answer is display: flex; justify-content: center; align-items: center — centered on both axes regardless of size. The classic gotcha is align-content appearing to do nothing on a single-line container: it only distributes space between wrapped lines, so it has no effect unless flex-wrap: wrap has actually produced more than one line.

.center {
  display: flex;
  justify-content: center; /* main axis  */
  align-items: center;     /* cross axis */
  min-height: 100vh;
}

gap and common flex patterns

gap (with its row-gap / column-gap longhands) adds consistent gutters between flex items without per-child margins or the old :last-child margin cleanup — it arrived in flexbox after grid but is now widely supported. With the container and item properties in hand, a small set of patterns covers most UI: nav bars (space-between, or an auto margin to push one item to the end), sticky card footers (a column container plus margin-top: auto on the footer), the media object, and equal-height columns (which you get free from the default align-items: stretch).

Why it matters: interviewers love "build a nav bar" or "push this button to the right" precisely because they reveal whether you hold gap, auto margins, and the axis model together. Drilling these patterns until they are reflexive pays off more than memorizing every property name.

.nav {
  display: flex;
  align-items: center;
  gap: 1.5rem;
}
.nav .cta { margin-inline-start: auto; } /* push to the far end */

Interview questions

Flexbox is one of the most-asked CSS layout topics in Indian front-end interviews — from service-company screening rounds to product-company machine-coding — so expect both quick definitions and "build or fix this layout" follow-ups:

Beginner What is flexbox and what problem does it solve?

A one-dimensional layout mode (display: flex). It distributes space and aligns a row or column of items whose sizes may be unknown or dynamic — replacing the old float, inline-block and vertical-align hacks for component layout. Grid is its two-dimensional counterpart.

Beginner How do you create a flex container, and what becomes a flex item?

Set display: flex (block-level container) or inline-flex on an element. Its direct children automatically become flex items laid along the main axis; grandchildren are unaffected unless they too become containers. Bare text runs among the children form anonymous flex items.

Beginner What is the difference between the main axis and the cross axis?

The main axis is the direction flex-direction sets (row → horizontal, column → vertical); items lay out along it. The cross axis is perpendicular. justify-content acts on the main axis, align-items on the cross axis — so switching flex-direction swaps which is which.

Beginner justify-content vs align-items?

justify-content distributes items along the main axis (space-between, center…); align-items aligns them on the cross axis (stretch default, center, baseline…). "Along the line" vs "across the line" — for a default row that reads as horizontal vs vertical.

Beginner How do you perfectly center an element with flexbox?

On the parent: display: flex; justify-content: center; align-items: center. That centers the child on both axes regardless of its size — the modern replacement for margin/transform centering tricks. Add a height (e.g. min-height: 100vh) so there is cross-axis space to center within.

Beginner What are the default values of flex-direction, flex-wrap, and a new flex item's flex?

flex-direction: row and flex-wrap: nowrap (so flex-flow: row nowrap). A flex item defaults to flex: 0 1 auto — it won't grow, will shrink, and is sized from its content or width. align-items defaults to stretch.

Intermediate What does flex: 1 expand to, and what does each part mean?

flex: 1 is flex-grow: 1; flex-shrink: 1; flex-basis: 0. Grow 1 = take an equal share of free space; shrink 1 = give up space equally under pressure; basis 0 = ignore content size when dividing, so items end up equal width. Contrast flex: auto (1 1 auto), which factors content size in first.

Intermediate flex-basis vs width — which wins?

On a flex item, flex-basis sets the initial main-size and takes precedence over width (when flex-basis: auto, it falls back to width; min-/max-width still clamp the result). Prefer flex-basis for the starting size and let grow/shrink adjust from there.

Intermediate flex-grow vs flex-shrink, and what does flex-shrink: 0 do?

flex-grow distributes positive free space (container larger than items); flex-shrink removes negative free space (items larger than container), each weighted by the item's factor. flex-shrink: 0 means "never shrink below my basis" — a frequent cause of overflow when a fixed-width item refuses to yield space.

Intermediate align-items vs align-content — why does align-content sometimes do nothing?

align-items aligns items within their line on the cross axis; align-content distributes the wrapped lines themselves. On a single-line container (flex-wrap: nowrap, the default) there is only one line, so align-content has no effect — it needs flex-wrap: wrap and actual wrapping to matter.

Intermediate What does the order property do, and what is the catch?

order (an integer, default 0) changes the visual sequence of flex items without moving them in the DOM; lower values come first. The catch is accessibility: screen-reader and keyboard/tab order follow the DOM, not order, so a large visual/DOM mismatch confuses users. Use it sparingly and keep the DOM in a sensible reading order.

Pro What does margin: auto do on a flex item, and do margins collapse inside a flex container?

An auto margin on a flex item absorbs all free space on that side: margin-left: auto (or margin-inline-start: auto) pushes an item and everything after it to the far end, and margin: auto centers a single item on both axes. Margins do not collapse in a flex container — flex items don't participate in block-margin collapsing, so adjacent item margins add rather than merge.

Pro A flex item with long text or an image overflows the container even though items should shrink. Why, and how do you fix it?

Flex items have min-width: auto by default (min-height: auto in a column), which resolves to the content's min-content size — so an item won't shrink below a long unbroken word or an image's intrinsic width, forcing overflow. Fix: set min-width: 0 (or min-height: 0 for columns) on the item, or add overflow: hidden, so flex-shrink can actually reduce it. This is the classic "why does my flex layout overflow" senior question.
