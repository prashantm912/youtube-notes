Module: Pseudo-classes and Pseudo-elements

One colon selects elements in a particular state; two colons style parts of an element that were never in your HTML. Master both and you delete whole classes of extra markup, wrapper divs, and JavaScript event handlers.

Knowledge Library → Courses → CSS → Pseudo-classes and Pseudo-elements

What this module covers

Pseudo-classes match elements by state or position the DOM tree alone can't express — hovered, focused, the third child, an element that contains an image. Pseudo-elements let you style or generate a sub-part of an element — a list marker, the selected text, an inserted icon — without adding nodes. This module orients you on five families and links the official reference for each; the newer functional selectors (:is, :where, :has) and highlight pseudo-elements are the parts interviewers now probe.

Topics

State pseudo-classes (:hover/:focus/:focus-visible/:active/:target/:checked)

These match an element while it is in a user-driven or navigation state: :hover under the pointer, :active while pressed, :focus when focused, :target when its id matches the URL fragment, and :checked for a ticked checkbox or radio. They are the engine behind hover effects, keyboard focus rings, CSS-only tabs/accordions, and the "checkbox hack" — interactivity with no JavaScript.

Why it matters: the modern must-know is :focus-visible. It shows a focus ring only when the browser's heuristics say the user is navigating by keyboard, so you can keep an accessible indicator for keyboard users without a distracting outline on mouse clicks. Removing focus outlines without a :focus-visible replacement is a common accessibility failure and a frequent interview trap.

button:focus-visible { outline: 2px solid Highlight; outline-offset: 2px; }
/* keep the ring for keyboard users, drop it for mouse clicks */
button:focus:not(:focus-visible) { outline: none; }

Structural (:nth-child/:nth-of-type/:first/:last/:only/:empty)

Structural pseudo-classes select by position among siblings without any extra classes: :first-child, :last-child, :only-child, :empty (no children at all), and the workhorses :nth-child() and :nth-of-type(), which take an An+B formula or the keywords even/odd. Zebra-striped tables, "every third card", and "remove the last border" all come from these.

Why it matters: the classic gotcha is :nth-child vs :nth-of-type. :nth-child(2) matches an element only if it is literally the 2nd child of its parent (counting all siblings, any tag); :nth-of-type(2) counts only siblings of the same element type. Mixed content — a heading followed by paragraphs — is where the two diverge and where candidates trip. Selectors Level 4 also adds :nth-child(An+B of S) to count only elements matching selector S.

tbody tr:nth-child(even) { background: #f4f4f4; } /* zebra striping */
.grid > .card:nth-child(3n) { margin-inline-end: 0; } /* every 3rd card */
.list:empty { display: none; }                    /* hide when nothing inside */

Functional (:is/:where/:not/:has)

Functional pseudo-classes take a selector list as an argument. :is() and :where() collapse long comma lists — :is(h1, h2, h3) a instead of three rules — and use forgiving parsing, so an unknown selector in the list is ignored instead of killing the whole rule. :not() is negation. :has(), the relational selector, matches an element based on what it contains or is followed by — long wished-for as the "parent selector".

Why it matters: specificity is the interview hook. :where() always contributes zero specificity (ideal for low-priority defaults and design systems), while :is() and :not() take the specificity of their most specific argument. :has() unlocks patterns that used to need JavaScript — styling a label when its input is checked, a form when a field is invalid, a card when it holds an image.

/* extra padding on any card that contains an image */
.card:has(img) { padding-block: 1rem; }
/* highlight a form group whose control is invalid */
.field:has(:invalid) { border-inline-start: 3px solid crimson; }
/* zero-specificity defaults, easy to override */
:where(.prose a) { color: royalblue; }

Form pseudo-classes (:valid/:invalid/:required/:disabled/:popover-open)

Form pseudo-classes reflect the live state of controls: :required/:optional, :valid/:invalid (from HTML constraint validation), :disabled/:enabled, :read-only/:read-write, and :checked. They let CSS style validation feedback with no JavaScript — a red border on an invalid email, a required-field asterisk, a dimmed disabled button. The newer :popover-open matches a popover (an element with the popover attribute) while it is showing, pairing with the Popover API.

Why it matters: the standard pitfall is that :invalid matches from the moment the page loads, so empty required fields look "wrong" before the user has typed anything. The fixes — :user-invalid, gating the rule so it applies only after the field is no longer empty (see the snippet below), or a "was submitted" class — are exactly what an interviewer wants to hear. Always pair color cues with text or an icon for accessibility.

input:required + label::after { content: " *"; color: crimson; }
input:invalid:not(:placeholder-shown) { border-color: crimson; }
:popover-open { border: 1px solid; padding: 1rem; }

Pseudo-elements (::before/::after/::marker/::selection/::placeholder/::backdrop/::part/::highlight)

Pseudo-elements style a part of an element that isn't a separate DOM node. ::before and ::after inject generated content (they need a content property, even content: "") for icons, decorations, and clearfix. Others target specific parts: ::marker (a list bullet/number), ::selection (highlighted text), the pseudo-element that styles an input's hint text, ::backdrop (the layer behind a dialog or fullscreen element), and ::part() (a named part exposed by a web component's shadow tree).

Why it matters: the double-colon signals "not a real element", and each pseudo-element accepts only a limited property set (for example ::marker and ::selection take color and a handful of text/font properties). Two are on the cutting edge: the CSS Custom Highlight API's ::highlight() styles arbitrary developer-defined text ranges — think find-in-page or a code editor's highlighting — via the JavaScript HighlightRegistry. And a key accessibility caveat: content added with ::before/::after is not reliably read by assistive tech, so never put essential information there.

.tag::before { content: "#"; color: gray; }
li::marker { color: teal; }
::selection { background: gold; color: black; }

Interview questions

Most-asked selector questions in Indian interviews — from front-end screening rounds to product-company deep dives on specificity:

Beginner What is the difference between a pseudo-class and a pseudo-element?

A pseudo-class (single colon, :hover) selects an element in a particular state or position. A pseudo-element (double colon, ::before) styles or creates a specific part of an element that isn't its own DOM node. The single vs double colon is the modern convention; old browsers accepted a single colon for the original four pseudo-elements.

Beginner What do :hover, :focus and :active do, and what is :focus-visible?

:hover matches while the pointer is over the element, :active while it is being pressed, :focus while it holds focus. :focus-visible matches focus only when the browser thinks a visible indicator is warranted (typically keyboard navigation) — so you keep an accessible focus ring for keyboard users without an outline flashing on every mouse click.

Beginner How does :nth-child(2n+1) work, and what do even/odd mean?

Substitute n = 0, 1, 2, … into An+B: 2n+1 yields 1, 3, 5, … so it matches the 1st, 3rd, 5th siblings — the same as odd. even equals 2n (2, 4, 6, …). Counting starts at 1 for the first child.

Beginner How do you style a required or invalid form field without JavaScript?

Use the form pseudo-classes: input:required, input:invalid, input:valid, input:disabled. These reflect HTML constraint validation live. Pair the color with text or an icon so the state isn't conveyed by color alone.

Intermediate :nth-child vs :nth-of-type — what's the difference?

:nth-child(n) counts every sibling regardless of tag, and matches only if the element at that index is of the type you selected. :nth-of-type(n) counts only siblings of the same type. In a list of identical <li> they behave the same; in mixed content (a heading then paragraphs), p:nth-child(1) can match nothing while p:nth-of-type(1) matches the first paragraph.

Intermediate :is() vs :where() — when would you choose each?

Both shorten selector lists and use forgiving parsing. The difference is specificity: :is() takes the specificity of its most specific argument, whereas :where() always has zero specificity. Use :where() for easily-overridable defaults and design-system base styles; use :is() when you want the selector to carry normal weight.

Intermediate How does :not() affect specificity, and what's a common gotcha?

:not() itself adds nothing; it contributes the specificity of its most specific argument. Gotchas: :not(.x) matches almost everything including <html>/<body>, and historically an invalid selector inside :not() invalidated the whole rule — wrap with :not(:is(...)) for forgiving behavior.

Intermediate What does ::before need to render, and why shouldn't you put important text in it?

It needs a content property — even content: "" — otherwise the pseudo-element isn't generated. Don't put essential information in ::before/::after because generated content is not reliably exposed to screen readers and can't be selected or copied; keep it decorative.

Intermediate Why does :invalid style empty fields on page load, and how do you fix it?

:invalid matches as soon as constraint validation fails, which is immediately for an empty required field — so fields look wrong before the user types. Fixes: use :user-invalid (matches only after interaction), gate the rule so it applies only once the field is no longer empty, or add a "submitted" class on the form and scope the styles to it.

Pro What is :has(), and give two things it enables that used to need JavaScript.

:has() is the relational pseudo-class: it matches an element if the relative selector inside matches something it contains or is followed by — the long-awaited "parent selector", though it does more. Examples: style a label:has(:checked) when its control is ticked, add spacing to a .card:has(img) that contains an image, flag a form:has(:invalid), or style a heading followed by a figure with h2:has(+ figure). Baseline across browsers since December 2023.

Pro Explain forgiving selector parsing and the :nth-child(An+B of S) syntax.

Forgiving parsing (used by :is()/:where()) means an unknown or unsupported selector in the argument list is dropped rather than invalidating the entire selector — great for progressive enhancement. :nth-child(An+B of S) applies the positional formula only to siblings matching selector S: e.g. li:nth-child(2n of .active) counts among .active items, which :nth-of-type can't express since it's keyed to element type, not an arbitrary selector.

Pro ::selection vs ::highlight() — what's the difference and what can you style?

::selection styles the user's current text selection; ::highlight() styles arbitrary developer-defined ranges registered in JavaScript via the CSS Custom Highlight API (HighlightRegistry and Highlight with AbstractRange objects) — used for find-in-page, spellcheck, or editor highlighting without wrapper spans. Both are highlight pseudo-elements and accept only a small property set: color, background-color, text-decoration, text-shadow, and the -webkit-text-stroke/fill properties. ::highlight() reached Baseline in 2026.
