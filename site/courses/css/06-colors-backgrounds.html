Module: Colors and Backgrounds

Colour and background are where a page gets its personality — and where wide-gamut spaces, color-mix() and light-dark() have quietly rewritten what an interviewer expects you to know.

Knowledge Library → Courses → CSS → Colors and Backgrounds

What this module covers

This module is the visual surface of CSS: every way to write a colour (from named keywords and hex through rgb()/hsl() to the modern perceptual spaces oklch()/oklab(), plus the color-mix() and light-dark() helpers), the background-* longhands and how multiple background layers stack, gradients as CSS-generated images, and the box-shadow/text-shadow pair. These are low-effort to start and surprisingly deep once modern colour and layering enter the conversation.

Topics

Color syntaxes (named/hex/rgb/hsl/hwb/lab/lch/oklab/oklch/color-mix/light-dark)

Anywhere CSS expects a <color> value you can supply one of a large family: keywords (named colours, transparent, currentColor), hex in 3-, 4-, 6- and 8-digit forms (the last two carry an alpha channel), the sRGB functions rgb() and hsl(), hwb(), and the wide-gamut, perceptually designed lab(), lch(), oklab() and oklch(). Sitting on top are two functional helpers: color-mix() blends two colours in a chosen colour space, and light-dark() returns one of two colours based on the active colour scheme.

Why it matters: for years hsl() was the "human" way to choose colours, but its lightness is not perceptually even — the same L across different hues can look far brighter or darker. oklch() fixes that, which is why design systems now generate colour ramps and accessible contrast pairs in it, and why interviewers increasingly ask why you would prefer it. The space-separated alpha syntax and light-dark() theming are the other two things that separate a current answer from a 2015 one.

:root { color-scheme: light dark; }
.btn {
  color: light-dark(#111, #eee);        /* one value, both themes */
  background: oklch(62% 0.19 255);      /* wide-gamut blue */
}
.btn:hover {
  background: color-mix(in oklch, oklch(62% 0.19 255), white 15%);
}

background-* longhands, multiple backgrounds

The background shorthand hides eight longhands — background-color, background-image, background-position, background-size, background-repeat, background-origin, background-clip and background-attachment. Setting them individually gives fine control; the shorthand is convenient but resets every longhand you omit back to its initial value — the classic "my background-color disappeared after I added an image" bug.

Why it matters: one element can carry several background layers at once, written as comma-separated images that stack with the first listed on top and only the last permitted a background-color. Layering a translucent gradient over a photo, or two gradients together, is how many "no image asset needed" UI effects are built. background-clip: text for gradient text and background-attachment: fixed for parallax-style effects are the details that surface in questions.

.hero {
  background:
    linear-gradient(rgb(0 0 0 / 60%), rgb(0 0 0 / 60%)),  /* top layer */
    url("photo.jpg") center / cover no-repeat;             /* back layer */
}

Gradients (linear/radial/conic/repeating)

Gradients are images generated by CSS, so they live anywhere an image can: background-image, border-image, mask and more. There are three shapes — linear-gradient() runs along a line or angle, radial-gradient() radiates from a point outward as a circle or ellipse, and conic-gradient() sweeps around a centre (the basis of pie charts and colour wheels). Each has a repeating-* variant that tiles its colour stops across the box.

Why it matters: the real skill is colour stops and hints. Placing two stops at the same position makes a hard edge — the trick behind stripes, flags and pie slices — and the interpolation colour space (for example in oklch) changes how the midtones look. Because a gradient is just an image, you can stack several with commas to build plaids and mesh-like effects with no asset at all.

.stripes {
  background: repeating-linear-gradient(
    45deg, #f3f3f3 0 10px, #e0e0e0 10px 20px);
}

box-shadow, text-shadow

box-shadow paints one or more shadows around an element's border box: offset-x, offset-y, an optional blur radius, an optional spread, a colour, and an optional inset keyword for inner shadows. text-shadow is the same idea for text and its decorations, but with no spread and no inset. Both accept a comma-separated list, so you can stack several shadows for depth or a glow.

Why it matters: two gotchas earn interview time. First, box-shadow follows the rectangular border box (rounded by border-radius) — to cast a shadow that hugs a transparent PNG or an irregular shape you need filter: drop-shadow() instead. Second, layered box-shadows are the standard way to build realistic "elevation" systems, and because animating a shadow can be expensive, people often fake the animation by fading a pseudo-element's opacity.

.card {
  box-shadow: 0 1px 2px rgb(0 0 0 / 0.06),
              0 8px 24px rgb(0 0 0 / 0.12);
}
.title { text-shadow: 0 1px 2px rgb(0 0 0 / 0.4); }

Interview questions

The colour and background questions that actually come up in Indian front-end and full-stack rounds — from the sRGB basics a screening round checks to the modern-CSS depth product companies push on:

Beginner What are the different ways to specify a colour in CSS?

Named keywords (plus transparent and currentColor), hex (#rgb, #rrggbb, and the 4-/8-digit forms with alpha), the sRGB functions rgb() and hsl(), hwb(), and the wide-gamut spaces lab(), lch(), oklab(), oklch(). Mention color-mix() and light-dark() as the modern helper functions.

Beginner What does the alpha channel do, and how do you set it in rgb() or hsl()?

Alpha is opacity of the colour itself, from fully transparent to fully opaque. Modern syntax is space-separated with a slash: rgb(0 0 0 / 60%) or hsl(210 50% 40% / 0.5). The old comma form rgba()/hsla() still works but is now just an alias.

Beginner What are the parts of a box-shadow value?

offset-x, offset-y, then optional blur radius, optional spread radius, a colour, and an optional inset keyword for an inner shadow. Only the two offsets are required; multiple shadows are comma-separated.

Beginner How do you place a background image so it covers the element without tiling?

background-image for the source, background-size: cover to fill while preserving aspect ratio, background-repeat: no-repeat to stop tiling, and background-position: center to keep the focal point centred. All can be written in one background shorthand.

Beginner How does currentColor help, and where is it useful?

currentColor resolves to the element's color value, so borders, SVG fills, shadows and gradients can track the text colour automatically. It is a cheap way to keep an icon or outline in sync with its label without repeating the colour.

Intermediate With the background shorthand, what happens to properties you don't list?

They reset to their initial values. The most common casualty is background-color: adding background: url(...) after a separate background-color wipes the colour. Fix it by including the colour in the shorthand or setting the longhands.

Intermediate How do multiple backgrounds layer, and which one is on top?

Comma-separated background layers stack with the first listed on top and each subsequent one behind it. Only the final layer may carry a background-color, since colour paints behind every image. This is how you put a dark gradient over a photo for readable overlay text.

Intermediate linear vs radial vs conic gradients — when would you reach for each?

linear for directional fades and stripes, radial for spotlights and soft circular glows, conic for anything sweeping around a centre — pie charts, colour wheels, progress rings. Each has a repeating-* form for patterns, and duplicate stop positions create hard edges.

Intermediate Why prefer oklch/oklab over hsl for generating a colour palette?

HSL's lightness is not perceptually uniform — the same L looks much brighter for yellow than for blue, so ramps built on it are uneven and contrast is unpredictable. Oklab/oklch are designed to be perceptually uniform, so stepping lightness or chroma gives visually even ramps and more reliable contrast, and the hue stays stable while you adjust.

Intermediate What is color-mix() and give a practical use.

color-mix(in <space>, colorA pct, colorB pct) blends two colours in a chosen colour space. Practical uses: deriving hover/active shades by mixing a brand colour with white or black, tinting, or building a whole theme from one token — color-mix(in oklch, var(--brand), white 20%).

Pro How does light-dark() work and what must be set for it to take effect?

light-dark(lightValue, darkValue) returns the first or second value based on the active colour scheme, so you get per-scheme colours in a single declaration without a @media (prefers-color-scheme) block. It only responds once color-scheme is declared (typically :root { color-scheme: light dark; }); otherwise it always resolves to the light value.

Pro box-shadow vs filter: drop-shadow() — what is the key difference?

box-shadow is cast by the rectangular border box (respecting border-radius), so a transparent PNG or clipped shape still throws a rectangular shadow. filter: drop-shadow() follows the actual alpha silhouette, so it hugs cut-outs and irregular shapes — but it has no spread or inset and, as a filter, can be more GPU-intensive and clips differently. Use box-shadow for solid boxes, drop-shadow for shaped or transparent graphics.
